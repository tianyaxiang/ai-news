# Anthropic’s Claude Tag is learning your company, one Slack message at a time
# Anthropic 的 Claude Tag 正在通过每一条 Slack 消息深入了解你的公司

Anthropic is introducing a new service called Claude Tag, an “always-on Claude” that lives in Slack and acts as an AI teammate. The new feature — which allows users to tag @Claude to provide insights in chats and assign tasks — will be available in beta for Slack for Claude Enterprise and Claude Team customers.

Anthropic 推出了一项名为 Claude Tag 的新服务，这是一个“全天候在线的 Claude”，它驻留在 Slack 中，充当 AI 队友。这项新功能允许用户通过 @Claude 来获取聊天中的见解并分配任务，目前已面向 Claude Enterprise 和 Claude Team 的 Slack 客户提供测试版。

Claude Tag is an evolution of several integrations that already exist. Users can already DM @Claude within Slack or tag it in channels for on-demand help, and Claude Code in Slack routes coding tasks from channel mentions to full coding sessions on the web, posting updates back into the thread. But Claude Tag adds a layer of persistent context and memory that would be difficult to maintain with previous tools.

Claude Tag 是现有多种集成功能的进化版。用户此前已经可以在 Slack 中私信 @Claude 或在频道中 @ 它以获取即时帮助，而 Slack 中的 Claude Code 则能将频道中提到的编码任务引导至网页端的完整编程会话，并将更新发布回对话线程中。但 Claude Tag 增加了一层持久的上下文和记忆功能，这是以往工具难以实现的。

“As Claude follows along with its channel, it learns ever more about the work,” reads a statement from Anthropic. “Claude can also automatically gather facts from elsewhere in the organization, if it’s granted permission to read other channels.”

Anthropic 在一份声明中表示：“随着 Claude 在频道中持续跟进，它对工作的了解会越来越深入。如果获得读取其他频道的权限，Claude 还可以自动从组织的其他地方收集信息。”

With Claude Tag, everyone in a given Slack channel can access a single Claude identity, meaning “anyone can see what Claude has been working on, and can pick up the conversation from where the last person left off.” System administrators will specify which tools, information, and channels Claude can access, and each Claude identity will stay scoped to whichever channels the admins define, so that a Claude set up for legal work can’t seed memories into the engineering channel, for example.

通过 Claude Tag，Slack 频道中的每个人都可以访问同一个 Claude 身份，这意味着“任何人都能看到 Claude 正在处理的工作，并能从上一个人中断的地方继续对话”。系统管理员将指定 Claude 可以访问哪些工具、信息和频道，每个 Claude 身份都将仅限于管理员定义的频道范围内，例如，为法律工作设置的 Claude 不会将记忆渗透到工程频道中。

When assigned a specific task, Claude Tag will break down the task into stages and will work through them using whichever tools it has access to, responding in a Slack thread with what it has created. But Claude Tag also features an ambient mode that proactively jumps into the chat of its own accord to keep your team updated, flag things from across the organization, and follow up on threads or tasks that have been forgotten.

当被分配特定任务时，Claude Tag 会将任务拆解为多个阶段，并利用其可访问的工具逐步完成，然后在 Slack 线程中回复其创建的内容。此外，Claude Tag 还具备一种“环境模式”，能够主动介入聊天，及时向团队通报进展、标记组织内的重要事项，并跟进那些被遗忘的线程或任务。

Anthropic says this makes it feel like you’re “working with a real colleague — one that can produce work in public view, with far greater context and understanding than before.” That context is an increasingly critical part of enterprise deployments, and Anthropic isn’t the only company focused on it. Microsoft also has Graph, expressed through Copilot and Work IQ. Snowflake and Databricks are positioning their platforms as the back-end support containing tacit organizational knowledge that agents can tap into. Glean is also building an intelligence layer that understands company context and sits between the model and the enterprise data.

Anthropic 表示，这让你感觉就像是“与一位真正的同事共事——一位可以在公开视野下产出工作成果，且比以往拥有更强上下文理解能力的同事”。这种上下文能力在企业部署中正变得愈发关键，而 Anthropic 并非唯一关注此领域的公司。微软拥有通过 Copilot 和 Work IQ 呈现的 Graph；Snowflake 和 Databricks 则将各自的平台定位为包含隐性组织知识的后端支持，供 AI 代理调用；Glean 也在构建一个能够理解公司背景、位于模型与企业数据之间的智能层。
