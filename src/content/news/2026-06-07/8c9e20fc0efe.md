### OpenAI / Plugins

**Plugins**
This repository contains a curated collection of Codex plugin examples. Each plugin lives under `plugins/<name>/` with a required `.codex-plugin/plugin.json` manifest and optional companion surfaces such as `skills/`, `.app.json`, `.mcp.json`, plugin-level agents/, `commands/`, `hooks.json`, `assets/`, and other supporting files.

**插件**
本仓库包含了一系列精选的 Codex 插件示例。每个插件都位于 `plugins/<name>/` 目录下，并包含一个必需的 `.codex-plugin/plugin.json` 清单文件，以及可选的配套组件，例如 `skills/`、`.app.json`、`.mcp.json`、插件级 agents/、`commands/`、`hooks.json`、`assets/` 以及其他支持文件。

**Highlighted richer examples in this repo include:**
* **plugins/figma**: for use_figma, Code to Canvas, Code Connect, and design system rules
* **plugins/notion**: for planning, research, meetings, and knowledge capture
* **plugins/build-ios-apps**: for SwiftUI implementation, refactors, performance, and debugging
* **plugins/build-macos-apps**: for macOS SwiftUI/AppKit workflows, build/run/debug loops, and packaging guidance
* **plugins/build-web-apps**: for deployment, UI, payments, and database workflows
* **plugins/expo**: for Expo and React Native apps, SDK upgrades, EAS workflows, and Codex Run actions
* **plugins/netlify, plugins/remotion, and plugins/google-slides**: for additional public skill- and MCP-backed plugin bundles

**本仓库中重点展示的丰富示例包括：**
* **plugins/figma**：用于 use_figma、Code to Canvas、Code Connect 以及设计系统规则。
* **plugins/notion**：用于规划、研究、会议记录和知识获取。
* **plugins/build-ios-apps**：用于 SwiftUI 实现、重构、性能优化和调试。
* **plugins/build-macos-apps**：用于 macOS SwiftUI/AppKit 工作流、构建/运行/调试循环以及打包指导。
* **plugins/build-web-apps**：用于部署、UI、支付和数据库工作流。
* **plugins/expo**：用于 Expo 和 React Native 应用、SDK 升级、EAS 工作流以及 Codex Run 操作。
* **plugins/netlify、plugins/remotion 和 plugins/google-slides**：用于其他基于技能（skill）和 MCP 的公共插件包。
