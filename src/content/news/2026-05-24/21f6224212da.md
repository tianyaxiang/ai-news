**A new suite of modern tools coming for editing and publishing RFCs**
**一套用于编辑和发布 RFC 的现代化工具即将推出**

**RFC Editor IETF Tools Team | 12 May 2026**
**RFC 编辑与 IETF 工具团队 | 2026年5月12日**

After a number of years of work, the initial rollout of a modern suite of tools for editing and publishing RFCs, including an entirely new rfc-editor.org website, will take place in May 2026.
经过多年的努力，一套用于编辑和发布 RFC 的现代化工具套件（包括一个全新的 rfc-editor.org 网站）将于2026年5月首次推出。

**UPDATE 12 May 2026:** Launch of the new tools for editing and publishing RFCs originally anticipated for 13 May has been deferred until the week of 18 May.
**更新 2026年5月12日：** 原定于5月13日推出的 RFC 编辑与发布新工具，现已推迟至5月18日当周发布。

Since its inception in 1969 with the publication of RFC 1, the RFC Series has had a formal editing and management function that has evolved through multiple iterations, as the series has itself evolved, to the RFC Editor function that we have today.
自1969年 RFC 1 发布以来，RFC 系列一直拥有正式的编辑和管理职能。随着该系列的不断演进，这一职能也经历了多次迭代，最终形成了我们今天所见的 RFC 编辑职能。

Now with over 9000 documents in the series, RFCs are authored in a variety of tools, often tracked through formal version control, published in RFCXML, and rendered into multiple presentation formats. The role of the RFC Production Center (RPC), the team that delivers the RFC Editor function, has changed significantly as a result.
目前，该系列文档已超过9000份。RFC 文档通过多种工具编写，通常采用正式的版本控制进行跟踪，以 RFCXML 格式发布，并渲染成多种展示格式。因此，负责执行 RFC 编辑职能的 RFC 生产中心（RPC）团队的角色也发生了重大变化。

In recent years it became clear that the tools used by the RPC could not adapt as needed to support these changes. The underlying database was too old, too rigid and too fragile, the editing tools were a large collection of unrelated and hard to maintain scripts, and the rfc-editor.org website was built on a platform far too old to add new functionality and improve usability. In particular, it was clear that the current tools could not support the editing or publication of RFC 10,000 and beyond.
近年来，人们愈发清楚地认识到，RPC 所使用的工具已无法适应这些变化。其底层数据库过于陈旧、僵化且脆弱；编辑工具是一堆互不关联且难以维护的脚本；而 rfc-editor.org 网站所基于的平台也过于老旧，无法添加新功能或提升易用性。特别是，现有的工具显然无法支持 RFC 10,000 及后续文档的编辑或发布。

In mid-2022 the IETF Administration LLC began a project to completely rewrite all of the RPC tools and add the new functionality that authors and the community had long been asking for, comprising the following:
2022年中期，IETF 管理 LLC 启动了一个项目，旨在彻底重写所有 RPC 工具，并增加作者和社区长期以来一直要求的新功能，具体包括：

The RPC database and workflow system will be migrated into a new application tightly integrated with Datatracker and using the existing Datatracker database of RFCs. The work on this has included:
RPC 数据库和工作流系统将迁移到一个与 Datatracker 紧密集成的新应用程序中，并使用现有的 Datatracker RFC 数据库。相关工作包括：

*   Many hours of manually reconciling differences in data between the RPC and Datatracker databases to provide a single source of truth.
    *   耗费大量时间手动核对 RPC 和 Datatracker 数据库之间的数据差异，以提供单一事实来源。
*   Implementing a new flexible state management system to allow for RPC processes to quickly adapt to meet future needs. This was in direct response to the many editorial process changes that have either been hard to implement or taken years longer than expected. This also provides for full end-to-end measurement of the RFC editing process, which has not been possible before now, and higher granularity measurement of individual parts of the process.
    *   实施一套全新的灵活状态管理系统，使 RPC 流程能够快速适应未来需求。这是为了直接解决许多编辑流程变更难以实施或耗时过长的问题。此外，这也实现了对 RFC 编辑流程的全程端到端度量（此前无法做到），并能对流程的各个环节进行更高粒度的测量。
*   An authors’ (public) view of the editing status of every document in the publication queue through the new subsite queue.rfc-editor.org. A screenshot of the new RFC Editor queue website that provides improved display of document status.
    *   通过全新的子网站 queue.rfc-editor.org，为作者提供发布队列中每份文档的编辑状态（公开视图）。（此处附带了新 RFC Editor 队列网站的截图，展示了改进后的文档状态显示界面。）

The RFC Editor website will migrate to an entirely new site that is designed around the needs of RFC consumers and sets the scene for more advanced features to be added. The work on this has included:
RFC Editor 网站将迁移到一个全新的站点，该站点围绕 RFC 用户需求进行设计，并为未来添加更高级的功能奠定了基础。相关工作包括：

*   Using professional UX designers and accessibility specialists to completely redesign and test the website experience, and then implementing that in a modern web framework. The site will be a high-quality, authoritative and accessible site for RFCs.
    *   聘请专业的 UX 设计师和无障碍专家对网站体验进行全面重新设计与测试，并使用现代 Web 框架进行实现。该网站将成为一个高质量、权威且易于访问的 RFC 站点。
*   Connecting to the new database and front-end object stores through all new APIs to support five-digit RFCs and beyond, faster page delivery under heavy load, and more efficient bulk access.
    *   通过全新的 API 连接到新数据库和前端对象存储，以支持五位数及以上的 RFC 文档，实现高负载下的快速页面加载，并提供更高效的批量访问。
*   Implementing a new errata system at errata.rfc-editor.org. This is substantively the same as the existing system with improved screening of reports and new login integration with Datatracker.
    *   在 errata.rfc-editor.org 实施全新的勘误系统。该系统在实质上与现有系统相同，但改进了报告筛选机制，并与 Datatracker 实现了新的登录集成。
*   Implementing a new search system with both full text and enhanced metadata searching. The new www.rfc-editor.org website makes RFCs easier to find and read with improved search and accessibility for those with sight impairments, and is friendlier to use on mobile devices.
    *   实施一套支持全文搜索和增强元数据搜索的新搜索系统。全新的 www.rfc-editor.org 网站通过改进搜索功能和针对视障人士的无障碍访问，使 RFC 更易于查找和阅读，同时在移动设备上也更加友好。

The RPC will migrate to a new editing tool, DraftForge, a Visual Studio Code plugin that provides a one-stop user interface for all their editing tools.
RPC 将迁移到一款名为 DraftForge 的新编辑工具，这是一个 Visual Studio Code 插件，为其所有编辑工具提供了一站式的用户界面。
