Many data scientists spend much of their time working with tabular data. Traditionally, these workloads have run on CPUs, which is often sufficient until an expensive operation becomes a bottleneck. Meanwhile, much of the discussion around GPUs focuses on deep learning and large language models. Today, however, GPU acceleration is available across much of the Python data science stack, allowing many existing tabular data workflows to run on GPUs with little or no code changes.

### Accelerating Pandas Workflows on GPU
### 在 GPU 上加速 Pandas 工作流

If your data processing pipeline is built with pandas, there are two easy ways to run it on the GPU:
如果你的数据处理流水线是基于 pandas 构建的，有两种简单的方法可以在 GPU 上运行它：

1. Working directly with the native cuDF API.
2. Accelerating existing pandas code with cudf.pandas.
1. 直接使用原生的 cuDF API。
2. 使用 cudf.pandas 加速现有的 pandas 代码。

Let’s look at both these approaches.
让我们看看这两种方法。

#### 1. Building GPU-native workflows with the cuDF API
#### 1. 使用 cuDF API 构建 GPU 原生工作流

If you’re starting a new project and want to work directly with GPU DataFrames, cuDF could be a great option. It is part of RAPIDS, NVIDIA’s open-source suite of GPU-accelerated libraries for data science and analytics. cuDF is a Python GPU DataFrame library for manipulating tabular data using a pandas-like API. So, if you’re familiar with pandas, much of the syntax will look familiar. The difference, however, is that unlike pandas, cuDF executes these operations on the GPU rather than the CPU.
如果你正在启动一个新项目并希望直接使用 GPU DataFrame，cuDF 是一个很好的选择。它是 RAPIDS 的一部分，这是 NVIDIA 为数据科学和分析提供的开源 GPU 加速库套件。cuDF 是一个 Python GPU DataFrame 库，使用类似 pandas 的 API 来操作表格数据。因此，如果你熟悉 pandas，大部分语法看起来会很眼熟。不同之处在于，与 pandas 不同，cuDF 在 GPU 而非 CPU 上执行这些操作。

#### Installation
#### 安装

如果你正在使用 Google Colab，cuDF 已在 GPU 运行时中可用。否则，请安装与你已安装的 CUDA 版本相匹配的 cuDF 包。你可以通过运行 `!nvidia-smi` 来检查你的 CUDA 版本。

#### Loading data into a cuDF DataFrame
#### 将数据加载到 cuDF DataFrame 中

Loading data into a cuDF DataFrame works just like pandas. In most cases, you simply replace `pandas.read_*` with the corresponding `cudf.read_*` function. Here’s how you’d load a Parquet file using cuDF:
将数据加载到 cuDF DataFrame 的方式与 pandas 完全相同。在大多数情况下，你只需将 `pandas.read_*` 替换为相应的 `cudf.read_*` 函数。以下是如何使用 cuDF 加载 Parquet 文件：

```python
import cudf
taxi_url = 'https://d37ci6vzurychx.cloudfront.net/trip-data/yellow_tripdata_2023-01.parquet'
df_gpu = cudf.read_parquet(taxi_url)
df_gpu.info()
```

Once the data is loaded into GPU memory, subsequent DataFrame operations run on the GPU.
一旦数据加载到 GPU 内存中，后续的 DataFrame 操作就会在 GPU 上运行。

#### Working with cuDF
#### 使用 cuDF

cuDF supports many of the same operations you already use in pandas. Here are a few common examples:
cuDF 支持许多你已经在 pandas 中使用的相同操作。以下是一些常见示例：

*   **Selecting columns:** `trips = df_gpu[["PULocationID", "DOLocationID", "fare_amount", "trip_distance", "tip_amount"]]`
*   **Filtering rows:** `high_fare = trips[trips["fare_amount"] > 20]`
*   **Creating new columns:** `trips["tip_percentage"] = (trips["tip_amount"] / trips["fare_amount"] * 100)`
*   **Summary statistics:** `trips.describe()`

*   **选择列：** `trips = df_gpu[["PULocationID", "DOLocationID", "fare_amount", "trip_distance", "tip_amount"]]`
*   **过滤行：** `high_fare = trips[trips["fare_amount"] > 20]`
*   **创建新列：** `trips["tip_percentage"] = (trips["tip_amount"] / trips["fare_amount"] * 100)`
*   **汇总统计：** `trips.describe()`

#### Moving between pandas and cuDF
#### 在 pandas 和 cuDF 之间转换

Many data science workflows use libraries that only support pandas. You can easily move data between CPU and GPU DataFrames using cuDF, when required.
许多数据科学工作流使用的库仅支持 pandas。在需要时，你可以使用 cuDF 在 CPU 和 GPU DataFrame 之间轻松移动数据。

*   **Converting a pandas DataFrame to cuDF:** `df_gpu = cudf.from_pandas(df_cpu)`
*   **Converting a cuDF DataFrame back to pandas:** `df_cpu = df_gpu.to_pandas()`

*   **将 pandas DataFrame 转换为 cuDF：** `df_gpu = cudf.from_pandas(df_cpu)`
*   **将 cuDF DataFrame 转换回 pandas：** `df_cpu = df_gpu.to_pandas()`

These conversions are useful when a downstream library only supports pandas or when you want to migrate an existing workflow to the GPU.
当下游库仅支持 pandas，或者当你想要将现有工作流迁移到 GPU 时，这些转换非常有用。
