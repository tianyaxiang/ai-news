# How to Refactor Code with Claude Code
# 如何使用 Claude Code 重构代码

LLM Applications: Improve coding agent productiveness with refactored code.
LLM 应用：通过重构代码提升编程代理的生产力。

In this article, I’ll discuss how to read your codebases using Claude Code. Claude Code and other coding agents are amazing at quickly implementing a lot of code. However, if you’ve ever worked a lot with coding agents, you’ll notice they start struggling after a while.
在本文中，我将探讨如何使用 Claude Code 来阅读和处理你的代码库。Claude Code 和其他编程代理在快速实现大量代码方面表现出色。然而，如果你曾大量使用过编程代理，你会发现它们在一段时间后会开始变得吃力。

Maybe you’ve spent a few days writing code with AI. Everything went perfectly and fast, and then you start noticing the tasks take longer, and the AI is making more and more mistakes. This is a clear sign that you need to start refactoring the code.
也许你已经花了好几天时间用 AI 编写代码。起初一切进展得既完美又迅速，但随后你开始注意到任务耗时变长，且 AI 犯错的频率越来越高。这是一个明确的信号，表明你需要开始重构代码了。

I believe that refactoring your codebase is going to be standard after you’ve implemented a lot of code with coding agents. Coding agents do make mistakes and don’t write perfect code. Also, if you don’t follow best practices in existing code, the AI will continue not following best practices in the new code it generates. Thus, you need to continuously refactor your repositories.
我相信，在使用编程代理实现大量代码后，重构代码库将成为一种标准流程。编程代理确实会犯错，写出的代码也并非完美。此外，如果你在现有代码中不遵循最佳实践，AI 在生成新代码时也会继续沿用这些不规范的做法。因此，你需要持续地重构你的代码仓库。

### Why refactor your code?
### 为什么要重构代码？

First of all, I always like to cover why you should care about the topic I’m discussing. Refactoring the code sounds like a boring task, and in reality, it used to be a very boring task that you had to do every now and then. Just a few years ago, humans had to manually go into the code repository to look for what needed to be refactored and start refactoring the code.
首先，我总是喜欢先说明为什么要关注我所讨论的话题。重构代码听起来是一项枯燥的任务，实际上，它过去确实是一项需要时不时进行的乏味工作。就在几年前，人类还必须手动进入代码仓库，寻找需要重构的部分并开始手动修改。

This would typically happen once, just after a lot of implementations in the code, because code naturally drifts away from a perfect state into a messier state simply by people interacting with the codebase and working on it. This happens even if humans are working on it or if AI agents are working on the codebase.
这种情况通常发生在代码经过大量实现之后，因为代码在人们与其交互和工作的过程中，会自然地从理想状态逐渐变得混乱。无论是由人类还是 AI 代理在处理代码库，这种情况都会发生。

You need to refactor your code because if the code is messy, it simply takes more time to do implementations, and you’re more prone to errors. Let me give a specific example. Let’s say we have a chatbot design implementation in your application where users can write to an AI bot and get a response back.
你需要重构代码，因为如果代码混乱，实现功能所需的时间会增加，且更容易出错。让我举个具体的例子。假设你的应用程序中有一个聊天机器人设计，用户可以向 AI 机器人发送消息并获得回复。

A good code base would ensure that you have a single component that’s called `chat`, and this component is used in all the different places in the app where you display the chat. A poorly designed code base would have one version of this chat per different location in the app. The problem then occurs once you want to make a change to the interface of the chat.
一个优秀的代码库会确保你拥有一个名为 `chat` 的单一组件，并在应用中所有需要显示聊天的地方复用它。而设计糟糕的代码库则会在应用的不同位置分别维护该聊天功能的不同版本。当你想要更改聊天界面时，问题就出现了。

### When to refactor your codebase
### 何时重构你的代码库

To put it simply, you should refactor your code when the coding agents are becoming slower at making implementations and when you start noticing the agents creating more bugs during an implementation. If you see issues appearing in unexpected places, or in code you didn’t touch, it is a clear sign that you should start refactoring.
简单来说，当编程代理实现功能的速度变慢，或者你开始注意到代理在实现过程中制造了更多 Bug 时，你就应该重构代码了。如果你发现问题出现在意想不到的地方，或者出现在你并未触碰的代码中，这就是需要开始重构的明确信号。

Furthermore, I recommend you start refactoring earlier rather than later. Of course, working with code that needs refactoring takes more time and is more prone to errors, but it’s also very frustrating because you’ll notice that you start making changes, and the coding agents are not able to do exactly what you tell them to do.
此外，我建议尽早开始重构。当然，处理需要重构的代码会耗费更多时间且更容易出错，但更令人沮丧的是，你会发现当你开始进行修改时，编程代理无法完全按照你的指令行事。

### 如何使用 Claude Code 重构代码

既然你已经知道何时应该重构代码库，我将开始讨论如何使用 Claude Code 或你偏好的任何编程代理来执行重构。每当我开始重构时，我都会遵循几个简单的规则：使用最先进、推理能力最强的模型。
