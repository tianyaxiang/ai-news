#### 1. Escaneo dinámico y menú interactivo
#### 1. 动态扫描与交互式菜单

Para evitar errores de tipeo al introducir el nombre de la imagen .wim, el script barre el directorio donde se almacenan y las lista en un menú. La decisión de hacerlo así fue para que sea escalable y dinámico permitiendo agregar imágenes al directorio sin necesidad de modificar el script.

为了避免输入 .wim 镜像名称时出现拼写错误，脚本会扫描存储镜像的目录并将其列在菜单中。这样做的目的是为了实现可扩展性和动态性，允许在目录中添加镜像而无需修改脚本。

```batch
:: Escaneo dinámico de imágenes
set COUNT=0
for %%A in ("%IMG_DIR%\*.wim") do (
    set /a COUNT+=1
    set "WIM[!COUNT!]=%%~fA"
    set "WIM_NAME[!COUNT!]=%%~nA"
)
:: ... generación del menú interactivo ...
```

Este simple bucle `for` detecta los archivos de un directorio y almacena las rutas en "arrays" simulados de Batch, generando opciones numeradas para que solo deba seleccionar una y el script se encargará del resto. Nota de sintaxis: El uso de signos de exclamación (!) en lugar del porcentaje (%) tradicional de Batch se debe a la necesidad de habilitar la expansión retardada (`setlocal enabledelayedexpansion`). Sin esto, Batch evaluaría las variables del menú al cargar el bucle en memoria y no en tiempo de ejecución, lo que rompería la lógica del índice dinámico.

这个简单的 `for` 循环会检测目录中的文件，并将路径存储在 Batch 模拟的“数组”中，生成带编号的选项，这样我只需选择一个，脚本就会处理剩下的工作。语法说明：使用感叹号 (!) 而不是传统的百分号 (%) 是因为需要启用延迟变量扩展 (`setlocal enabledelayedexpansion`)。如果没有它，Batch 会在将循环加载到内存时评估菜单变量，而不是在运行时评估，这会破坏动态索引的逻辑。

#### 2. Integración de DiskPart al vuelo
#### 2. 实时集成 DiskPart

En lugar de depender de un archivo de texto externo para particionar el disco, el propio script genera las instrucciones de diskpart en el directorio temporal y las ejecuta en segundo plano:

脚本不再依赖外部文本文件来对磁盘进行分区，而是直接在临时目录中生成 diskpart 指令并在后台执行：

```batch
set "TEMP_SCRIPT=%temp%\dp_script.txt"
(
    echo select disk %TARGET_DISK%
    echo clean
    echo convert gpt
    echo create partition efi size=200
    echo format quick fs=fat32 label="EFI"
    echo assign letter=S
    :: ... creación de particiones MSR y Windows ...
) > "%TEMP_SCRIPT%"
diskpart /s "%TEMP_SCRIPT%" >> "%LOG_FILE%" 2>&1
del "%TEMP_SCRIPT%"
```

Lo ejecutamos silenciosamente con `diskpart /s` y registramos su salida en un archivo de log (`%LOG_FILE%`) para diagnosticar posibles fallos.

我们使用 `diskpart /s` 静默执行它，并将输出记录到日志文件 (`%LOG_FILE%`) 中，以便诊断可能出现的故障。

#### 3. Configuración automática de arranque UEFI
#### 3. 自动配置 UEFI 引导

Una vez que DISM vuelca la imagen seleccionada, el script ejecuta el paso final para que el sistema sea arrancable de forma automática sin intervención manual:

一旦 DISM 部署完选定的镜像，脚本就会执行最后一步，使系统能够自动引导，无需人工干预：

```batch
bcdboot Z:\Windows /s S: /f UEFI >> "%LOG_FILE%" 2>&1
```

Este comando escribe los archivos de arranque en la partición EFI (S:) apuntando a la instalación de Windows que acabamos de aplicar en la partición primaria (Z:). Al concentrar todo en un único archivo de script, logramos un proceso autocontenido: el mismo archivo prepara el hardware, automatiza la selección de la imagen y configura el arranque UEFI.

该命令将引导文件写入 EFI 分区 (S:)，并指向我们刚刚在主分区 (Z:) 上应用的 Windows 安装。通过将所有内容集中在一个脚本文件中，我们实现了一个自包含的流程：同一个文件既准备了硬件，又自动化了镜像选择，还配置了 UEFI 引导。
