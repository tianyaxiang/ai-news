### GitHub is investigating unauthorized access to their internal repositories
### GitHub 正在调查其内部存储库遭未经授权访问的事件

GitHub has confirmed that it is currently investigating reports of unauthorized access to some of its internal repositories. The company stated that it is working to identify the scope of the incident and determine whether any sensitive data or customer information has been compromised.
GitHub 已证实，目前正在调查有关其部分内部存储库遭未经授权访问的报告。该公司表示，正在努力确定此次事件的影响范围，并核实是否有任何敏感数据或客户信息遭到泄露。

Security researchers first flagged the suspicious activity, noting that unauthorized parties may have gained access to private code repositories. GitHub has since initiated an internal investigation and is implementing additional security measures to prevent further unauthorized access.
安全研究人员率先发现了这一可疑活动，并指出未经授权的第三方可能已经访问了私有代码存储库。GitHub 随后启动了内部调查，并正在采取额外的安全措施，以防止进一步的未经授权访问。

"We are aware of the situation and are taking it very seriously," a GitHub spokesperson said. "We are committed to protecting our users' data and will provide updates as we learn more about the nature of this incident."
GitHub 发言人表示：“我们已经知悉相关情况，并对此高度重视。我们致力于保护用户数据，并将在进一步了解事件性质后提供最新进展。”

Users are advised to monitor their accounts for any unusual activity and to enable two-factor authentication (2FA) if they have not already done so. GitHub will notify any affected customers directly if their data is found to be involved in the breach.
建议用户密切关注账户是否存在异常活动，并尽快启用双重身份验证（2FA）。如果发现有客户的数据受到此次泄露事件的影响，GitHub 将直接通知相关用户。
