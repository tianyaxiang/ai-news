### UCCI: Calibrated Uncertainty for Cost-Optimal LLM Cascade Routing
### UCCI：用于成本最优大模型级联路由的校准不确定性方法

**Abstract:** LLM cascades and model routing promise lower inference cost by sending easy queries to a small model and escalating hard ones to a large model, but most deployed routers use uncalibrated confidence scores and require per-workload threshold tuning. 
**摘要：** 大模型（LLM）级联和模型路由通过将简单查询发送至小型模型、将复杂查询升级至大型模型，有望降低推理成本。然而，目前大多数已部署的路由方案使用未经校准的置信度分数，且需要针对每个工作负载进行阈值调整。

We present UCCI, a calibration-first router that maps token-level margin uncertainty to a per-query error probability via isotonic regression and selects the escalation threshold by constrained cost minimization. 
我们提出了 UCCI，这是一种以校准为先的路由方案。它通过保序回归（isotonic regression）将 token 级别的边际不确定性映射为每个查询的错误概率，并通过约束成本最小化来选择升级阈值。

Under three explicit assumptions, threshold policies on the calibrated score are cost-optimal, and isotonic calibration achieves O(n^{-1/3}) sample complexity for expected calibration error (ECE). 
在三个明确的假设下，基于校准分数的阈值策略是成本最优的，且保序校准在预期校准误差（ECE）方面实现了 O(n^{-1/3}) 的样本复杂度。

On a production named entity recognition workload of 75,000 queries served by 4B and 12B instruction-tuned LLMs on H100 GPUs, UCCI cuts inference cost by 31% (95% CI: [27%, 35%]) at micro-F1 = 0.91 while reducing ECE from 0.12 to 0.03. 
在 H100 GPU 上运行的 4B 和 12B 指令微调大模型处理 75,000 条命名实体识别（NER）生产查询的工作负载中，UCCI 在 micro-F1 达到 0.91 的前提下，将推理成本降低了 31%（95% 置信区间：[27%, 35%]），同时将 ECE 从 0.12 降低至 0.03。

At the same operating point, UCCI beats entropy thresholding, split-conformal routing, and a FrugalGPT-style learned threshold. 
在相同的运行指标下，UCCI 的表现优于熵阈值法、拆分共形路由（split-conformal routing）以及 FrugalGPT 风格的学习阈值法。

All cascade results use end-to-end routing on actual model outputs and measured H100 latency, not simulated routing from global accuracies or nominal API prices.
所有级联结果均基于实际模型输出的端到端路由以及实测的 H100 延迟，而非基于全局准确率或标称 API 价格的模拟路由。
