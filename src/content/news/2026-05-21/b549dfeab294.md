### First: Why Gemma 4 Is Different
### 首先：Gemma 4 有何不同？

Most open-source models are powerful but come with a catch - they either need the cloud (OpenAI, Anthropic), require massive hardware (LLaMA 70B), or sacrifice too much quality for local use (older 2B/7B models). Gemma 4 breaks that tradeoff. The 31B model currently ranks #3 among all open models globally on the Arena AI leaderboard. The 26B MoE ranks #6. Both run on consumer hardware. Both are Apache 2.0 licensed - meaning you can use them commercially, modify them, and deploy them anywhere, for free. But more importantly: Gemma 4 processes text, images, video, and audio natively. Not bolted-on, not a separate pipeline - natively, in a single model call. That changes what's possible to build locally.

大多数开源模型虽然强大，但都有局限性——要么依赖云端（如 OpenAI、Anthropic），要么需要庞大的硬件支持（如 LLaMA 70B），要么为了本地运行而牺牲了太多质量（如旧款 2B/7B 模型）。Gemma 4 打破了这种权衡。31B 模型目前在 Arena AI 全球排行榜上名列开源模型第三，26B MoE 模型排名第六。两者均可在消费级硬件上运行，且均采用 Apache 2.0 许可——这意味着你可以免费将其用于商业用途、进行修改并部署到任何地方。更重要的是：Gemma 4 原生支持处理文本、图像、视频和音频。这不是外挂功能，也不是独立的流水线，而是通过单次模型调用原生完成。这彻底改变了本地构建应用的可能性。

### Decoding the Model Names: E2B, E4B, 26B A4B, 31B
### 解码模型名称：E2B、E4B、26B A4B、31B

This is the part that confuses everyone. Let's decode it once and for all. The "E" Models: Edge. E2B = Effective 2 Billion parameters; E4B = Effective 4 Billion parameters. The word "effective" is doing real work here. These aren't traditional dense models - they use architectural tricks (Per-Layer Embeddings) to punch above their weight class. The "E" signals they're built for edge deployment: phones, Raspberry Pi, laptops, embedded hardware. Both E2B and E4B support: Text + Image + Audio input natively, up to 128K token context window, fully offline via Ollama, and no GPU required (CPU inference possible).

这是最让人困惑的部分。让我们彻底搞清楚它。以“E”开头的模型代表“边缘（Edge）”：E2B = 有效参数量 20 亿；E4B = 有效参数量 40 亿。“有效（Effective）”这个词在这里至关重要。它们不是传统的稠密模型，而是利用了架构技巧（逐层嵌入，Per-Layer Embeddings）来实现越级表现。“E”代表它们专为边缘部署而生：手机、树莓派、笔记本电脑和嵌入式硬件。E2B 和 E4B 均支持：原生文本+图像+音频输入、高达 128K 的上下文窗口、通过 Ollama 完全离线运行，且无需 GPU（支持 CPU 推理）。

The Workstation Models: 26B and 31B. 26B A4B = 26 Billion total parameters, 4 Billion Active per inference; 31B = 31 Billion Dense parameters. The 26B A4B uses a Mixture of Experts (MoE) architecture — it has 26B parameters but only activates 4B of them for any given token. This means it runs at roughly the speed of a 4B model while producing quality closer to a full 26B model. That "A" stands for Active. Both workstation models support: Text + Image + Video (up to 60 seconds at 1fps) input, up to 256K token context window, and hybrid attention (local sliding window + full global).

工作站模型：26B 和 31B。26B A4B = 总参数 260 亿，每次推理激活 40 亿；31B = 310 亿稠密参数。26B A4B 使用混合专家（MoE）架构——它拥有 260 亿参数，但对于每个 token 只激活其中的 40 亿。这意味着它的运行速度大致相当于 4B 模型，但输出质量却接近完整的 26B 模型。“A”代表“激活（Active）”。两款工作站模型均支持：文本+图像+视频（最长 60 秒，1fps）输入、高达 256K 的上下文窗口，以及混合注意力机制（局部滑动窗口 + 全局注意力）。

### The Architecture That Makes It Work
### 实现这一切的架构

**Hybrid Attention: Fast AND Deep**
Traditional transformers use full attention - every token attends to every other token. This is expensive. Gemma 4 uses a hybrid attention mechanism that interleaves: Local sliding window attention for nearby tokens (fast, memory-efficient) and Full global attention for the final layer (captures long-range dependencies). This gives you the processing speed of a lightweight model without losing the deep contextual understanding you need for complex tasks. The final layer is always global — ensuring nothing important gets missed.

**混合注意力机制：既快又深**
传统的 Transformer 使用全注意力机制——每个 token 都要关注其他所有 token，这非常昂贵。Gemma 4 使用了一种混合注意力机制，交替使用：针对附近 token 的局部滑动窗口注意力（快速、内存高效）和针对最后一层的全局注意力（捕捉长距离依赖）。这让你在获得轻量级模型处理速度的同时，不会丢失复杂任务所需的深度上下文理解。最后一层始终是全局的，确保不会遗漏任何重要信息。

**Proportional RoPE (p-RoPE)**
For long contexts (up to 256K tokens), Gemma 4 uses Proportional RoPE — a smarter positional encoding that scales proportionally with sequence length. Combined with unified Keys and Values in global attention layers, this dramatically reduces memory usage at long contexts compared to standard transformers.

**比例旋转位置编码 (p-RoPE)**
对于长上下文（高达 256K token），Gemma 4 使用了比例 RoPE——这是一种更智能的位置编码，随序列长度成比例缩放。结合全局注意力层中统一的键（Keys）和值（Values），与标准 Transformer 相比，这显著降低了长上下文下的内存占用。

**Variable-Resolution Vision**
When you send an image to Gemma 4, it doesn't resize everything to a fixed resolution. It uses a dynamic number of soft tokens fitted exactly to the image's content and resolution. A high-detail photo gets more tokens than a simple diagram. This matters enormously for real-world tasks like document analysis, medical imaging, or - in my case - crop disease detection where fine texture details are diagnostic signals.

**可变分辨率视觉**
当你向 Gemma 4 发送图像时，它不会将所有内容调整为固定分辨率。它使用动态数量的软 token，精确适配图像的内容和分辨率。高细节照片分配的 token 比简单图表更多。这对于文档分析、医学影像或我所做的作物病害检测等现实任务至关重要，因为精细的纹理细节往往是诊断信号。

**Per-Layer Embeddings (PLE) in Edge Models**
The E2B and E4B models use Per-Layer Embeddings instead of shared embeddings across all layers. This lets them carry more representational richness per parameter - which is why they're called "Effective" rather than just "2B" and "4B". They genuinely outperform models of equivalent raw parameter count.

**边缘模型中的逐层嵌入 (PLE)**
E2B 和 E4B 模型使用逐层嵌入，而不是在所有层之间共享嵌入。这使它们在每个参数上承载了更丰富的表征能力——这就是为什么它们被称为“有效（Effective）”而不仅仅是“2B”和“4B”。它们确实在性能上超越了同等原始参数规模的模型。

### The Thinking Mode: Built-in Reasoning
### 思维模式：内置推理

Every Gemma 4 model has a configurable thinking mode - the model can reason step-by-step before producing its final answer. In Ollama, you enable it like this:
```python
response = ollama.chat(
    model="gemma4:e4b",
    messages=[{"role": "user", "content": "your prompt"}],
    options={"thinking": True} # enables step-by-step reasoning
)
```
This is particularly powerful for: Diagnosis tasks (reasoning through symptoms before concluding), Code debugging (thinking through the logic before suggesting a fix), Multi-step math (working through the problem before giving an answer), and Agentic workflows (planning before acting). The thinking tokens are separate from the response — you get clean output plus the reasoning chain if you want to inspect it.
