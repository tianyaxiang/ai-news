🛡️ Local API CSRF hardening: a malicious web page can no longer drive unsafe cross-site requests (POST/PUT/DELETE) against the loopback API — CORS blocks reading the response but not the side effect, so loopback dev-mode trust now applies the existing cross-site guard to unsafe methods before honoring it. Safe methods and local CLI / non-browser uploads are unaffected (#293, thanks @Hinotoi-agent).

**2026-06-23**
🛡️ 本地 API CSRF 加固：恶意网页现在无法再针对回环（loopback）API 发起不安全的跨站请求（POST/PUT/DELETE）—— CORS 虽然阻止了读取响应，但无法阻止副作用，因此回环开发模式的信任机制现在会在执行前对不安全的方法应用现有的跨站防护。安全方法以及本地 CLI / 非浏览器上传不受影响（#293，感谢 @Hinotoi-agent）。
