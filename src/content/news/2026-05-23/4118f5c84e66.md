# Predicting Performance of Symbolic and Prompt Programs with Examples
# 基于示例预测符号程序与提示词程序的性能

LLM prompting is widely used for naturally stated tasks, yet it is unreliable; it may succeed on a few test cases but fail at deployment time. We study performance prediction: given a program, either symbolic (e.g. Python) or a prompt executed on an LLM, and a few in-domain examples, predict its performance on unseen tasks from the same domain.

大语言模型（LLM）提示词工程被广泛用于处理自然语言描述的任务，但其可靠性较差——它可能在少数测试用例上成功，却在实际部署时失败。我们研究了性能预测问题：给定一个程序（无论是符号程序如 Python，还是在 LLM 上执行的提示词程序）以及少量领域内示例，预测其在同一领域未见任务上的性能。

We use a simple coin-flip model, treating each pass/fail program execution as a Bernoulli random variable, whose success probability is the program's unknown performance. In this model, performance depends entirely on: 1) the observed execution outcomes on test cases, and 2) a prior over performances.

我们使用了一个简单的抛硬币模型，将程序的每次执行（通过/失败）视为一个伯努利随机变量，其成功概率即为程序未知的性能。在该模型中，性能完全取决于：1）在测试用例上观察到的执行结果，以及 2）关于性能的先验分布。

We compile empirical performance priors from a corpus of diverse programs and tasks, and find that performance for symbolic programs (e.g., Python) are all or nothing, while prompt programs have a diffuse prior with many nearly-correct programs. This difference explains why a few passing tests can certify symbolic programs but not prompt programs.

我们从一个包含多种程序和任务的语料库中汇编了经验性能先验，发现符号程序（如 Python）的性能表现为“全或无”（要么完全正确，要么完全错误），而提示词程序则具有分散的先验分布，存在许多“接近正确”的程序。这一差异解释了为什么少数通过的测试用例可以验证符号程序，却无法验证提示词程序。

Building on this insight, we develop RAP (Retrieved Approximate Prior), which retrieves similar tasks and prompt programs from an existing corpus to construct a proxy prior, which is then used to predict performance. We show RAP achieves solid performances.

基于这一洞察，我们开发了 RAP（检索近似先验），它通过从现有语料库中检索相似的任务和提示词程序来构建代理先验，进而用于预测性能。实验表明，RAP 能够实现稳健的性能预测。
