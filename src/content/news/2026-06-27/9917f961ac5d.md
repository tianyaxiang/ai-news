# GeMoE: Gating Entropy is All You Need for Uncertainty-aware Adaptive Routing in MoE-based Large Vision-Language Models

**GeMoE：门控熵即一切——面向混合专家模型（MoE）大型视觉语言模型的感知不确定性自适应路由**

***

**Abstract:** With the increase in model parameters and training data, the instruction following and generalization capabilities of Large Vision-Language Models (LVLMs) have been significantly improved. Based on the Mixture of Experts (MoE) architecture, LVLMs expand their parameter capacity while maintaining the inference cost. However, traditional MoE methods employ a Top-k static routing strategy, which fails to account for variations in the input and adaptively select the number of experts, resulting in suboptimal resource utilization.

**摘要：** 随着模型参数和训练数据的增加，大型视觉语言模型（LVLMs）的指令遵循和泛化能力得到了显著提升。基于混合专家（MoE）架构，LVLMs 在保持推理成本的同时扩展了参数容量。然而，传统的 MoE 方法采用 Top-k 静态路由策略，无法根据输入的变化自适应地选择专家数量，导致资源利用率不理想。

***

In this paper, we propose viewing token routing as an information encoding task, framing dynamic routing as a Minimum Description Length (MDL) problem in encoding. By validating the connection between MDL and gating entropy in the MoE scenario, we introduce Gating Entropy-based Uncertainty-aware Adaptive Routing (GeMoE) for MoE. Unlike traditional static or heuristic-based dynamic routing methods, GeMoE explicitly models the trade-off between model complexity and performance.

在本文中，我们提出将 Token 路由视为一项信息编码任务，并将动态路由构建为编码中的最小描述长度（MDL）问题。通过验证 MoE 场景下 MDL 与门控熵之间的联系，我们引入了面向 MoE 的基于门控熵的感知不确定性自适应路由（GeMoE）。与传统的静态或基于启发式的动态路由方法不同，GeMoE 明确地对模型复杂度和性能之间的权衡进行了建模。

***

By using gating entropy to assess the complexity of tokens, GeMoE adaptively determines the number of experts each token should engage. On a wide range of backbones and benchmarks, our method achieves 99.5% average performance retention compared to the original static routing, while improving average expert activation sparsity by 36.5%.

通过利用门控熵来评估 Token 的复杂度，GeMoE 能够自适应地确定每个 Token 应激活的专家数量。在广泛的主干网络和基准测试中，我们的方法在保持与原始静态路由相比 99.5% 平均性能的同时，将平均专家激活稀疏度提高了 36.5%。
