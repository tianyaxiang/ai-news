### Convert images to vectors
### 将图像转换为向量

The first step is to convert images into vectors, which are numerical representations of visual data. Vector size is critical and the optimum size depends on the application. The lengths of 128, 512, or 768 dimensions are common choices. When we increase the size, we capture more information and expect more accurate results but it comes at the cost of larger storage size and possibly more latency in search.
第一步是将图像转换为向量，即视觉数据的数值表示。向量大小至关重要，最佳尺寸取决于具体应用。128、512 或 768 维是常见的选择。当我们增加维度时，可以捕获更多信息并期望获得更准确的结果，但代价是更大的存储空间占用以及可能更高的搜索延迟。

We need an embedding model to convert images to vectors. We can train our own model but there are several ready-to-use models available, both free and paid. For example, the following code block takes a JPEG image and converts it into a 512-dimensional vector using the open-source clip-ViT-B-32 model.
我们需要一个嵌入模型将图像转换为向量。我们可以训练自己的模型，但目前有许多现成的模型可用，包括免费和付费的。例如，以下代码块使用开源的 `clip-ViT-B-32` 模型，将一张 JPEG 图像转换为 512 维的向量。

```python
from PIL import Image
from sentence_transformers import SentenceTransformer

sku_image = Image.open("sample_image.jpeg")
model = SentenceTransformer('clip-ViT-B-32')
image_vector = model.encode(sku_image)

type(image_vector), image_vector.shape
# (numpy.ndarray, (512,))
```

The `image_vector` variable is a Numpy array of size 512.
`image_vector` 变量是一个大小为 512 的 Numpy 数组。

### Create a Milvus Collection
### 创建 Milvus 集合

Milvus is a vector database and a collection in Milvus is a two-dimensional table with fixed columns and rows. Each column represents a field, and each row represents an entity, which is an image in our case. We will create a collection with two fields: an id field (such as a product SKU) and a corresponding vector field for that SKU’s image.
Milvus 是一个向量数据库，其中的“集合”（Collection）是一个具有固定列和行的二维表。每一列代表一个字段，每一行代表一个实体（在我们的案例中即为图像）。我们将创建一个包含两个字段的集合：一个 ID 字段（如产品 SKU）和该 SKU 对应图像的向量字段。

There are several ways to create and interact with a collection. I prefer to use Python whenever I can so I’ll go with the `pymilvus` module. The first step is to create a client object. This is the way to connect to your vector database in Milvus.
创建和操作集合有多种方式。我倾向于尽可能使用 Python，因此我将使用 `pymilvus` 模块。第一步是创建一个客户端对象，这是连接到 Milvus 向量数据库的方式。

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient(uri="http://....") # your milvus db uri
```

Then, we define a schema for our collection:
然后，我们为集合定义一个模式（Schema）：

```python
schema = client.create_schema(auto_id=False, enable_dynamic_field=True)

# Add fields to schema
schema.add_field(field_name="sku_id", datatype=DataType.VARCHAR, max_length=512, is_primary=True)
schema.add_field(field_name="image_vector", datatype=DataType.FLOAT_VECTOR, dim=512)
```

The schema has two fields: `sku_id` and `image_vector`. Then, we can create the collection using the `create_collection` method:
该模式有两个字段：`sku_id` 和 `image_vector`。接下来，我们可以使用 `create_collection` 方法创建集合：

```python
collection_name = "test_collection"
client.create_collection(
    collection_name=collection_name,
    schema=schema,
    index_params=index_params
)
```

We can now add an index to our fields. An index is crucial for a Milvus collection, or any vector database. It accelerates search speeds and reduces query latency, especially when working with large vector datasets. One option to add index is to use the `create_index` function.
现在我们可以为字段添加索引。索引对于 Milvus 集合或任何向量数据库都至关重要。它能加速搜索速度并降低查询延迟，特别是在处理大规模向量数据集时。添加索引的一种方法是使用 `create_index` 函数。

```python
index_params = client.prepare_index_params()

# index for the image vector
index_params.add_index(
    field_name="image_vector",
    index_name="image_vector_idx",
    index_type="IVF_FLAT",
    metric_type="COSINE", # Using COSINE similarity (common for images). Can also be L2 or IP.
)

# index for "sku_id" (primary key)
index_params.add_index(
    field_name="sku_id",
    index_name="sku_id_idx",
    index_type="INVERTED"
)

collection_name = "test_collection"
client.create_index(
    collection_name=collection_name,
    index_params=index_params
)
```

In the final step, we load the collection and check its status.
最后一步，我们加载集合并检查其状态。

```python
client.load_collection(collection_name=collection_name)

# check load status
res = client.get_load_state(
    collection_name=collection_name
)
print(res)
# {'state': <LoadState: Loaded>}
```

We have created the collection, but it is currently empty. To confirm that our new collection was successfully created, we can list all collections in the database using the `list_collections` method.
我们已经创建了集合，但它目前是空的。为了确认新集合已成功创建，我们可以使用 `list_collections` 方法列出数据库中的所有集合。

```python
client.list_collections()
# ['test_collection']
```

The next step is to insert entities (i.e. skus and image vectors).
下一步是插入实体（即 SKU 和图像向量）。

### Insert Entities
### 插入实体

We can now load data into our collection. To do this, the data must be formatted as a dictionary, where the keys match the collection’s field names. We can then use a list of these dictionaries to insert multiple entities at once.
现在我们可以将数据加载到集合中。为此，数据必须格式化为字典，其中键名需与集合的字段名匹配。然后，我们可以使用这些字典的列表一次性插入多个实体。
