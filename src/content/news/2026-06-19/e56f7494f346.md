# Structured Outputs with LLMs: JSON Mode, Function Calling, and When to Use Each
# 大语言模型的结构化输出：JSON 模式、函数调用及其适用场景

In my latest posts, we’ve talked a lot about popular techniques for optimizing the performance and cost of AI applications, like response streaming or prompt caching. Today, I want to talk about something a bit different but equally important for building real AI apps. That is, structured, machine-readable outputs.
在我最近的文章中，我们讨论了许多优化 AI 应用性能和成本的常用技术，例如响应流（response streaming）或提示词缓存（prompt caching）。今天，我想谈谈一个略有不同但对于构建真实 AI 应用同样重要的话题：结构化、机器可读的输出。

So far in most of the examples I’ve shared, we’ve been dealing with free-text responses from an AI model. The user asks a question, the model responds in natural language, and we just display that response to the user in some way. Fairly simple and straightforward. But what happens when we need the model to return data in a specific format (e.g., a JSON object) so that we can further process it programmatically later on?
到目前为止，在我分享的大多数示例中，我们处理的都是 AI 模型生成的自由文本响应。用户提问，模型用自然语言回答，我们只需以某种方式将该响应展示给用户。这非常简单直接。但如果我们要求模型以特定格式（例如 JSON 对象）返回数据，以便后续进行程序化处理，该怎么办呢？

What if we need the model to extract specific fields from a text or image, populate a database entry, or trigger a subsequent action based on its response? In those cases, getting back a wall of text won’t be very convenient. 🤔
如果我们要求模型从文本或图像中提取特定字段、填充数据库条目，或根据其响应触发后续操作，又该怎么办？在这种情况下，返回一大段文本将非常不便。🤔

Happily, there are multiple solutions for this issue. There are two main approaches for obtaining structured, machine-readable outputs from an LLM: JSON Mode and Function Calling (also called tool use). These two are often confused with one another (which is to be expected since they both deal with structured outputs, duh), but they serve quite different purposes.
幸运的是，针对这个问题有多种解决方案。从大语言模型（LLM）获取结构化、机器可读输出主要有两种方法：JSON 模式和函数调用（也称为工具使用）。这两者经常被混淆（考虑到它们都处理结构化输出，这很正常），但它们的用途截然不同。

On top of this, OpenAI has introduced a stricter variant of Function Calling called Structured Outputs, which takes schema enforcement one step further, as we’ll see. In this post, we’ll take a closer look at all three, understand how each one works under the hood, and figure out when to use each. So, let’s take a look!
此外，OpenAI 还引入了一种更严格的函数调用变体，称为“结构化输出”（Structured Outputs）。正如我们将要看到的，它将模式强制执行（schema enforcement）提升到了一个新的高度。在本文中，我们将深入探讨这三种方法，了解它们各自的底层工作原理，并弄清楚何时使用哪种方法。让我们开始吧！

### 1. What is JSON Mode?
### 1. 什么是 JSON 模式？

JSON Mode is the simpler approach for achieving machine-readable outputs from an LLM. It is essentially a parameter you can set in an API request to instruct the model to always return a valid JSON object. And that’s really all there is to it!
JSON 模式是从 LLM 获取机器可读输出的更简单方法。它本质上是一个可以在 API 请求中设置的参数，用于指示模型始终返回有效的 JSON 对象。仅此而已！

Nonetheless, this simplicity comes at a cost, since there are no guarantees on the structure or schema of the JSON (remember we didn’t define any schema, field names, or types, or anything like this), just that it will be valid, parseable JSON.
然而，这种简单性是有代价的，因为无法保证 JSON 的结构或模式（请记住，我们没有定义任何模式、字段名称、类型或类似的东西），只能保证它是一个有效的、可解析的 JSON。

For example, using OpenAI’s API in Python, we can enable JSON Mode by adding the parameter `response_format={"type": "json_object"}` to our call to the model.
例如，在使用 Python 调用 OpenAI API 时，我们可以通过在模型调用中添加 `response_format={"type": "json_object"}` 参数来启用 JSON 模式。

If we run the same example multiple times, we may get slightly different field names or a slightly different structure each time. For example, one run might return `"name"`, and another `"full_name"`. That’s a problem if we’re trying to reliably extract specific fields programmatically.
如果我们多次运行同一个示例，每次可能会得到略有不同的字段名称或结构。例如，一次运行可能返回 `"name"`，另一次可能返回 `"full_name"`。如果我们试图通过程序可靠地提取特定字段，这就会成为一个问题。

### 2. What is Function Calling?
### 2. 什么是函数调用？

Function Calling (or tool use) is a more advanced approach for getting structured, machine-readable outputs from an LLM. Instead of just asking the model to format its response as JSON, we define a specific schema. That is, we explicitly define a formal description of the structure we want the output to follow, and in this way, the model is more constrained to return data that matches that schema exactly.
函数调用（或工具使用）是从 LLM 获取结构化、机器可读输出的更高级方法。我们不再仅仅要求模型将响应格式化为 JSON，而是定义一个特定的模式。也就是说，我们明确定义了我们希望输出遵循的结构描述，通过这种方式，模型被更严格地约束，以返回完全符合该模式的数据。

In other words, with Function Calling we define upfront what fields we expect, what types those fields should be, which are required, and which are not, and so on.
换句话说，通过函数调用，我们预先定义了我们期望的字段、这些字段的类型、哪些是必需的、哪些不是，等等。
