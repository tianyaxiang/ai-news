尽管该模型目前无法使用，但我相信并希望它能很快重新上线。我也认为，几个月后我们将看到其他具备同样能力的模型。当然，已经有很多文章在讨论 Claude Fable 及其能力。我相信本文的独特之处在于，我每天都在使用 Claude Code，并且自 2026 年初以来几乎从未间断。

Claude Fable 发布时，我立即对其进行了全面测试。我有一系列 Opus 无法一次性或轻松完成的任务，我用这些任务来真正考验 Claude Fable 的能力。你可以阅读本文来了解我对 Claude Fable 能力的看法，特别是它与 Claude Opus 4.8 和 GPT-5.5 等其他前沿模型的对比。

### Why care about Claude Fable 5
### 为什么要关注 Claude Fable 5？

First of all, we should start with why you should care about Claude Fable 5. This is perhaps the most anticipated LLM ever, as it had a lot of hype going around it for months before it was launched. Anthropic itself spent a lot of time hyping up the model, talking about its capabilities and how dangerous it could be in the wrong hands. A lot of people were thus excited for the launch of the model, and it finally launched last week with full access to anyone with a Claude Pro or Max subscription.
首先，我们应该从为什么要关注 Claude Fable 5 说起。这可能是史上最受期待的大语言模型，在发布前的几个月里，它就引发了巨大的炒作。Anthropic 本身也花费了大量时间为该模型造势，谈论其能力以及它在不当使用下可能带来的危险。因此，许多人对该模型的发布感到兴奋，它最终于上周发布，并向所有拥有 Claude Pro 或 Max 订阅的用户全面开放。

Personally, I did not experience any issues using the model until Saturday morning, Norwegian time. Even though the model was not available for too long (I believe around 72 hours), I feel like I got to test the model extensively and formed a good opinion on both its upsides, downsides, and overall capabilities.
就我个人而言，直到挪威时间周六上午，我在使用该模型时都没有遇到任何问题。尽管该模型可用的时间不长（大约 72 小时），但我感觉我已经对其进行了充分测试，并对其优势、劣势和整体能力形成了良好的判断。

### What Claude Fable does well
### Claude Fable 的优势

First of all, I want to cover what Claude Fable did well. My overall impression is that it’s significantly better than Claude Opus 4.8. I have read other people online mentioning how they felt that the capabilities didn’t go that much further than Claude Opus 4.8. In my opinion, this was obviously not true. I believe that people reporting this have not tested Claude Fable on complex enough tasks. Of course, if you try Claude Fable on an already easy task that Claude Opus is able to do, you’ll not experience its full capabilities. Where Claude Fable really shines is when you apply it to super complex coding tasks.
首先，我想谈谈 Claude Fable 的出色之处。我的总体印象是，它明显优于 Claude Opus 4.8。我看到网上有人提到，他们觉得其能力并没有比 Claude Opus 4.8 提升多少。在我看来，这显然是不真实的。我相信那些这样说的人并没有在足够复杂的任务上测试过 Claude Fable。当然，如果你只是用 Claude Fable 处理 Claude Opus 已经能完成的简单任务，你将无法体验到它的全部实力。Claude Fable 真正闪光的地方在于处理超复杂的编程任务。

### Completing complex tasks end-to-end
### 端到端完成复杂任务

I had several tasks I was working on where I had spent some time with Claude Opus 4.8 to implement it. Opus was definitely able to implement it, but it wasn’t a one-shot implementation, and I had to manually guide Claude Opus through some of the implementations. These tasks were, for example: A feature implementation where I had to change code in multiple repos; Fixing an issue/bug that a customer experienced, where I had to improve the information extraction capabilities of an LLM.
我手头有几个任务，之前花了一些时间用 Claude Opus 4.8 来实现。Opus 确实能够实现，但无法一次性完成，我必须手动引导 Claude Opus 完成部分实现过程。这些任务例如：需要在多个代码库中更改代码的功能实现；修复客户遇到的问题/漏洞，这需要改进大语言模型的信息提取能力。

What I did to compare Claude Opus versus Claude Fable is that I had previously applied Claude Opus to these problems and solved them with quite a bit of manual direction. I then applied it to Claude Fable, and it was able to one-shot the problems. A clear sign that Fable is a more powerful model than Opus.
为了对比 Claude Opus 和 Claude Fable，我之前曾用 Claude Opus 处理过这些问题，并在大量手动指导下才得以解决。随后我用 Claude Fable 尝试，它能够一次性解决这些问题。这清楚地表明 Fable 是比 Opus 更强大的模型。

As a more generic note on Fable’s capabilities, I would say the following: Claude Fable is more able to complete tasks end-to-end, with both better understanding of ambiguities and user intent, better at implementing the actual planned solution, and better at verifying the solution is actually correct through navigating on the computer or running integration tests.
关于 Fable 能力的更普遍评价，我想说：Claude Fable 更能端到端地完成任务，它对歧义和用户意图的理解更好，在执行实际计划方案时更出色，并且通过在计算机上导航或运行集成测试来验证方案正确性的能力也更强。

### Discovering issues in codebases
### 在代码库中发现问题

Another incredible capability I noticed in Claude Fable was that it was much better than Claude Opus at discovering issues in codebases, either finding bugs or looking for refactoring opportunities or potential future issues. I constantly run a prompt similar to the one below to discover issues in my codebase: "Scan thoroughly through the codebase to identify any potential bugs, issues, or refactoring opportunities, and come back to me with an HTML report with issues prioritized from most severe to least severe."
我在 Claude Fable 中注意到的另一个惊人能力是，它在发现代码库问题方面远胜于 Claude Opus，无论是查找漏洞、寻找重构机会还是发现潜在的未来问题。我经常运行类似于下面的提示词来发现代码库中的问题：“彻底扫描代码库以识别任何潜在的漏洞、问题或重构机会，并向我提供一份 HTML 报告，将问题按严重程度从高到低排序。”

With Claude Opus, I ran this same prompt and didn’t get good results either. Claude Opus wasn’t able to discover any more refactoring opportunities or bugs, or the issues it discovered weren’t really relevant.
使用 Claude Opus 时，我运行了同样的提示词，但没有得到好的结果。Claude Opus 无法发现更多的重构机会或漏洞，或者它发现的问题并不相关。
