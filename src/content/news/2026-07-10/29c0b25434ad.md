# How to Find the Optimal Coding Agent Interface
# 如何找到最佳的编程智能体（Coding Agent）交互界面

The interface that you use to interact with your coding agents is very important. There is a large variety of options out there that you can try, and they all impact how effectively you’re able to interact with your agents. I’ve spent a lot of time testing out different platforms to orchestrate coding agents, and in this article, I’ll give my opinion on some different tools and how you can find the tool that works best for you.

你与编程智能体交互时所使用的界面至关重要。市面上有各种各样的选择供你尝试，它们都会影响你与智能体交互的效率。我花了很多时间测试不同的平台来编排编程智能体，在本文中，我将分享我对一些不同工具的看法，以及你如何找到最适合自己的工具。

I think it’s worth noting that the optimal tool will vary from person to person. A tool that I like might be a tool that you dislike, and vice versa. Thus, I’ll also cover how to find out for yourself what works for you in this article. I’m not sponsored by any of the tools I cover in this article. These are simply tools that I personally find work well.

值得注意的是，最佳工具因人而异。我喜欢的工具你可能不喜欢，反之亦然。因此，我也会在本文中介绍如何自行判断什么工具适合你。本文中提到的所有工具均未向我提供赞助，这些仅仅是我个人认为表现良好的工具。

### Why spend time finding an optimal agent interface?
### 为什么要花时间寻找最佳的智能体界面？

First of all, I always like to cover the why in each article that I write. Here I wanted to highlight why you should spend time at all testing out different interfaces to interact with your coding agents. The main reason you should spend time doing this is that it can be more productive in the long run. If you can find an interface that works super well for you, it will make it easier for you to interact with your coding agents, and you’ll become more productive.

首先，我喜欢在每篇文章中阐述“为什么”。在这里，我想强调为什么要花时间去测试不同的界面来与编程智能体交互。你应该这样做的主要原因是，从长远来看，这能提高生产力。如果你能找到一个非常适合你的界面，它会让你与智能体的交互变得更轻松，从而提高你的工作效率。

I strongly believe there is a pretty stark difference between the tool I like the least and the tool that I like the most, and it strongly impacts how much I’m able to do. One of the main issues I find with some tools is that they are not easy to have a full overview of all the agents that I’m running at the same time. In this case, I have to spend time manually catching up or finding relevant sessions that I want to work on, which is, of course, wasted time that could be saved using a better tool.

我坚信，我最不喜欢的工具和我最喜欢的工具之间存在巨大差异，这极大地影响了我的工作产出。我发现某些工具的主要问题在于，无法轻松地全面概览我同时运行的所有智能体。在这种情况下，我必须花时间手动跟进或寻找我想要处理的相关会话，这显然是浪费时间，而使用更好的工具本可以节省这些时间。

### Different coding agent interfaces
### 不同的编程智能体界面

In this section, I’ll go through some different interfaces for coding agents, covering some different applications. The applications that I’ve tried for coding agents are: Warp, Conductor, Emdash, Iterm2, Claude Code application, Codex application, Omnara, and Cursor.

在本节中，我将介绍一些用于编程智能体的不同界面，涵盖几种不同的应用程序。我尝试过的编程智能体应用包括：Warp、Conductor、Emdash、Iterm2、Claude Code 应用、Codex 应用、Omnara 和 Cursor。

**Warp**
This was my go-to terminal for quite a while, and I think Warp is overall pretty good. It’s a simple terminal with some AI features, such as autocomplete when you write commands, and you can also easily run AI models inside Warp. They also have automatic naming of different sessions and allow you to split tabs. One downside I noticed with Warp is that it is a very plain terminal and doesn’t really have any added features, and I did experience it lagging a bit even though my computer was not lagging; I have not experienced issues like this with other applications.

**Warp**
这曾是我很长一段时间的首选终端，我认为 Warp 总体上相当不错。它是一个带有 AI 功能的简单终端，例如在编写命令时提供自动补全，你也可以轻松地在 Warp 中运行 AI 模型。它们还具有不同会话的自动命名功能，并允许拆分标签页。我注意到 Warp 的一个缺点是它是一个非常基础的终端，并没有太多附加功能。尽管我的电脑运行正常，但我确实体验到它偶尔会有卡顿，而在其他应用程序中我没有遇到过类似问题。

**Conductor**
Conductor is a very nice beginner-friendly terminal. It’s a very clean setup. I think the way they organize the different agents that you’re running is very good. They organize it into backlog, in progress, in review, and done, and then allow you to archive tabs and bring them back if you want to. The only downside I noticed in Conductor is actually two things. It doesn’t allow you to split tabs, which is very bad, of course. And secondly, it doesn’t have complete feature parity with the coding agents that I prefer. One way I noticed this is that it doesn’t have the slash goal command available for Claude Code sessions. It only has it for Codex sessions. This was a deal-breaker for me, so I had to stop using Conductor just because of that.

**Conductor**
Conductor 是一个非常适合初学者的终端，设置非常简洁。我认为他们组织运行中智能体的方式非常好。他们将其分为“待办（backlog）”、“进行中（in progress）”、“审核中（in review）”和“已完成（done）”，并允许你归档标签页并在需要时恢复。我在 Conductor 中注意到的唯一缺点其实有两点：它不允许拆分标签页，这当然很糟糕；其次，它与我偏好的编程智能体在功能上不完全对等。我发现的一个例子是，它在 Claude Code 会话中不可用 `/goal` 命令，仅在 Codex 会话中可用。这对我来说是无法接受的，因此我不得不因此停止使用 Conductor。

**Emdash**
Emdash is the current application that I’m using, and it’s my favorite at the moment. Emdash basically has everything Conductor has, but it also has feature parity with all CLI coding agents because it literally runs the terminal inside the application, and it allows you to split panes. The only downside, the way I see it, is that Emdash doesn’t have the equally good organization of tabs on the left hand side where you see all of your coding agents, i.e., it doesn’t have the backlog/ in progress / in review / done organization that you get in Conductor, but it still has a relatively good way of organizing the different agent tabs.

**Emdash**
Emdash 是我目前正在使用的应用程序，也是我现在的最爱。Emdash 基本具备了 Conductor 的所有功能，但它与所有 CLI 编程智能体功能对等，因为它直接在应用程序内运行终端，并且允许你拆分面板。在我看来，唯一的缺点是 Emdash 在左侧标签页的组织上不如 Conductor 那样出色（即没有 Conductor 那种“待办/进行中/审核中/已完成”的分类），但它仍然提供了一种相对不错的组织不同智能体标签页的方式。

**iTerm2**
iTerm2 is also an okay option; it’s just a basic terminal where you can run your coding agents. However, I see no reason to; I stopped using a plain terminal for a reason: it’s just that while they work fine because coding agents are accessible through a CLI, there are no real features, and it doesn’t really organize the tabs that nicely for you. So I stopped using iTerm2.

**iTerm2**
iTerm2 也是一个尚可的选择；它只是一个基础终端，你可以在其中运行编程智能体。然而，我看不出使用它的理由。我停止使用普通终端是有原因的：虽然它们工作正常（因为编程智能体可以通过 CLI 访问），但它们没有任何真正的功能，而且无法很好地为你组织标签页。所以我停止使用了 iTerm2。

**Claude Code application**
The Claude Code application is quite good and very beginner-friendly. However, again, I don’t like the way they organize the agent tabs on the left-hand side and the sessions that are running. I just believe that there are applications that organize the different tabs way better, such as Emdash and Conductor. It’s worth noting that I do use the Claude Code application through my phone because I have remote control on. And this works very well, and I really like the Claude Code application in this way.

**Claude Code 应用**
Claude Code 应用相当不错，对初学者非常友好。然而，我依然不喜欢它在左侧组织智能体标签页和运行会话的方式。我认为像 Emdash 和 Conductor 这样的应用程序在组织标签页方面做得更好。值得一提的是，我确实通过手机使用 Claude Code 应用，因为我开启了远程控制功能。这种方式运行得非常好，我非常喜欢以这种方式使用 Claude Code 应用。

**Codex application**
Codex application is basically the same as the Claude Code application, and I see no real differences in either the pros or the cons.

**Codex 应用**
Codex 应用与 Claude Code 应用基本相同，我在优缺点方面没有看到任何实质性差异。

**Omnara**
Omnara was marketed as kind of an application that syncs really well between your phone and your computer. While it did sync the phone and PC well, I found the overall app a bit clunky to use, and I didn’t find it that intuitive. Seems to me that the design wasn’t fully up...

**Omnara**
Omnara 被宣传为一款在手机和电脑之间同步效果极佳的应用程序。虽然它确实能很好地同步手机和电脑，但我发现整个应用使用起来有点笨重，而且我觉得它不够直观。在我看来，它的设计似乎还不够完善……
