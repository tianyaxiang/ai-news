# Together for a healthier Clippy
# 共建更健康的 Clippy

July 6, 2026 · blyxyas on behalf of the Clippy team
2026 年 7 月 6 日 · blyxyas 代表 Clippy 团队

Clippy is a very loved piece of software, with the fortune of having many contributors willing to do high-quality pull requests with very needed features and bug-fixes. We are extremely thankful for every single contributor who not only took the time to learn about Clippy’s internals, but also felt comfortable enough to show their work and maintained communication in order to ship it. That’s why we’re addressing them with this health report of Clippy as an open source project.
Clippy 是一款备受喜爱的软件，我们很幸运拥有许多贡献者，他们愿意提交高质量的 Pull Request (PR)，带来急需的功能和错误修复。我们衷心感谢每一位贡献者，他们不仅花时间深入了解 Clippy 的内部机制，还愿意展示自己的工作成果并保持沟通，以确保代码顺利合并。正因如此，我们发布这份 Clippy 开源项目健康报告，以回应大家的关切。

The truth is that Clippy has a reviewing capacity problem, and we’ve been having it for the last few months. Unlike some other Rust teams, no one from the Clippy team is currently funded to work on Clippy, not even on a part-time basis. The burden of maintenance thus falls squarely into the maintainer’s free time, eventually leading to burnout. We hope that the newly established Rust Foundation Maintainers Fund will be able to help secure some funding for Clippy maintainers. If you'd like to help, consider donating to the fund.
事实是，Clippy 目前面临着代码审查能力不足的问题，这种情况已经持续了几个月。与其他一些 Rust 团队不同，Clippy 团队目前没有任何成员获得全职或兼职的资助。因此，维护工作的重担完全落在了维护者的业余时间上，这最终导致了倦怠。我们希望新成立的 Rust 基金会维护者基金（Rust Foundation Maintainers Fund）能够帮助 Clippy 的维护者获得一些资助。如果您愿意提供帮助，请考虑向该基金捐款。

That’s why we are launching an initiative in which contributors help themselves get their pull request reviewed and merged faster. What does this mean? Once a contributor opens a pull request, we propose they review another contributor’s pull request. That way each contributor reviews at least one pull request and nobody’s contributions get stale.
因此，我们发起了一项倡议，旨在让贡献者通过互助的方式，加快各自 PR 的审查和合并速度。这意味着什么呢？当贡献者提交一个 PR 时，我们建议他们去审查另一个贡献者的 PR。通过这种方式，每位贡献者至少会审查一个 PR，从而确保没有任何贡献被搁置。

The underlying mechanism looks like this: Once a contributor opens a pull request, our dear bot @rustbot will prompt them to look at other pull requests (preferably ones without any other reviews). After they’ve reviewed a pull request, the contributor gets on a special secret list of cool ones, and priority on their pull request from the team's reviewers. We want to thank the Bevy project and their "Open Code Review" system as an inspiration for this new initiative.
其底层机制如下：一旦贡献者提交了 PR，我们亲爱的机器人 @rustbot 就会提示他们去查看其他 PR（最好是那些尚未被审查的）。在完成审查后，该贡献者将被列入一份特别的“酷贡献者”名单，其 PR 也将获得团队审查人员的优先处理。我们要感谢 Bevy 项目及其“开放代码审查”（Open Code Review）系统，它们为我们这项新倡议提供了灵感。

We've done this in the past. Clippy is no stranger to health updates of this kind, last year we had the Feature Freeze, which was an absolute success. In that process, we gained knowledge, trust, and the certainty that the Rust community cared about Clippy, and Clippy doing things right.
我们过去也曾这样做过。Clippy 对这类健康状况更新并不陌生，去年我们实施了“功能冻结”（Feature Freeze），并取得了巨大成功。在那个过程中，我们获得了经验、信任，并确信 Rust 社区关心 Clippy，也关心 Clippy 是否在朝着正确的方向发展。

Once again, the team wants to thank and congratulate every single person, contributor or not, that cares about Clippy. Striving for excellence is what makes this linter (and language) so loved, and with the help of the community (of YOU), we can help Clippy become stronger and healthier.
最后，团队再次感谢并祝贺每一位关心 Clippy 的人，无论您是否是贡献者。追求卓越是这款代码检查工具（以及这门语言）如此受欢迎的原因，在社区（也就是你们）的帮助下，我们可以让 Clippy 变得更强大、更健康。
