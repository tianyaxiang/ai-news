# From Data Analyst to Data Engineer: My 12-Month Self-Study Roadmap
# 从数据分析师到数据工程师：我的 12 个月自学路线图

**Ibrahim Salami | May 16, 2026**

To be honest, a part of me started this journey because data engineering is one of the hottest and highest-paying careers right now. I’m not going to pretend that wasn’t a factor. But there’s more to it than that. I’ve been learning data analytics for a while now. SQL, Power BI, Python (Pandas, NumPy, a little Polars), data cleaning, EDA. You name it, I’ve been in the weeds with it. And I genuinely enjoy it.
老实说，我开启这段旅程的部分原因是因为数据工程是目前最热门、薪资最高的职业之一。我不会假装这不是一个考量因素。但除此之外，还有更深层的原因。我学习数据分析已经有一段时间了：SQL、Power BI、Python（Pandas、NumPy，还有一点 Polars）、数据清洗、探索性数据分析（EDA）。你能想到的，我都深入钻研过，而且我确实乐在其中。

But somewhere along the way, I started getting curious about what happens before the data lands on my desk. How does it move? Who builds those pipelines? What does the infrastructure behind all of this actually look like? That curiosity planted a seed. Then AI started making a lot of what I do faster and easier. Which is great. But it also made me think: if AI can handle the analysis, what’s my edge? What can I build and understand that goes deeper?
但在某个阶段，我开始好奇：在数据到达我的桌面之前，发生了什么？它是如何流动的？是谁构建了那些管道？这一切背后的基础设施到底是什么样的？这种好奇心埋下了一颗种子。随后，人工智能开始让我的许多工作变得更快、更简单。这固然很好，但也让我思考：如果 AI 能处理分析工作，我的优势在哪里？我能构建和理解什么更深层次的东西？

### Why Data Engineering Specifically
### 为什么选择数据工程？

I want to spend a moment here because I think this question deserves a real answer. Data analytics taught me how to work with data after it arrives. Clean it, explore it, visualize it, draw insights from it. That skillset is genuinely valuable. But the more I learned, the more I kept bumping into the same wall. The data I was working with had already been shaped and moved by someone else. Someone had built the pipeline that brought it to me. Someone had decided how it was stored, how it was structured, how often it refreshed. I wanted to be that person.
我想花点时间谈谈这个问题，因为它值得一个认真的回答。数据分析教会了我如何在数据到达后进行处理：清洗、探索、可视化并从中提取洞察。这些技能非常有价值。但随着学习的深入，我不断撞上同一堵墙：我处理的数据早已被他人塑造和迁移。有人构建了将数据带给我的管道，有人决定了数据如何存储、如何结构化以及更新频率。我想成为那个人。

Data engineering sits upstream from analytics. It’s about building the systems that make analysis possible in the first place. Data pipelines, storage architecture, workflow orchestration, large-scale data processing. These are the foundations everything else is built on. And honestly, that kind of infrastructure work appeals to me in a way that pure analysis no longer does.
数据工程位于分析的上游。它旨在构建使分析成为可能的基础系统。数据管道、存储架构、工作流编排、大规模数据处理——这些是构建其他一切事物的基石。老实说，这种基础设施工作对我的吸引力，已经超过了纯粹的分析工作。

### Why I’m Doing This in Public
### 为什么我要公开学习？

Writing about what I learn is something I already believe in deeply. It forces you to actually understand something before you explain it. It keeps you accountable. And over time, it builds something that a resume alone never could. But I’ll be honest about my fears too, because I think that’s the point of doing this publicly. I have shiny object syndrome. There, I said it. I’ve explored graphic design, animation, writing, marketing, and IT before landing in data. There’s always something new and exciting pulling my attention.
我深信记录学习过程的价值。它迫使你在解释某件事之前真正理解它，它能让你保持自律，久而久之，它能建立起一份简历无法比拟的个人品牌。但我也会坦诚我的恐惧，因为我认为这正是公开学习的意义所在。我有“闪光物体综合症”（容易被新鲜事物吸引）。没错，我承认了。在进入数据领域之前，我探索过平面设计、动画、写作、市场营销和 IT。总有新鲜刺激的事物在吸引我的注意力。

### What I’m Starting With
### 我的起点

I’m not starting from zero, which helps. I already have beginner to intermediate SQL knowledge from my data analytics work, basic Python fundamentals, and some hands-on experience with Pandas. That gives me a foundation to build on rather than rebuild from scratch. Here’s the full learning stack, roughly in the order I’ll be tackling it.
我并非从零开始，这很有帮助。得益于数据分析工作，我已经具备了初中级的 SQL 知识、Python 基础以及一些 Pandas 的实操经验。这为我提供了一个可以深耕的基础，而不是一切从头再来。以下是我完整的学习技术栈，大致按我学习的顺序排列：

1. **SQL: Going Deeper Than Analytics**
   **SQL：深入分析之外的领域**
   I know SQL. But analytics SQL and engineering SQL are different animals. I’ll be going deeper into query optimization, indexing, working with very large datasets, and writing SQL that’s built for performance rather than just exploration.
   我懂 SQL，但分析型 SQL 和工程型 SQL 是两码事。我将深入研究查询优化、索引、处理超大规模数据集，并编写以性能为导向而非仅用于探索的 SQL 代码。

2. **Python: From Exploratory to Production-Ready**
   **Python：从探索性代码到生产级代码**
   I have the basics. Pandas, NumPy, some Polars. But the Python I’ve been writing lives mostly in notebooks. Exploratory, messy, not built to last. The goal now is to write cleaner, more structured, reusable code. Functions, modules, error handling, scripting. The kind of Python you’d actually put in a pipeline.
   我掌握了基础知识：Pandas、NumPy、一些 Polars。但我之前写的 Python 代码大多存在于 Notebook 中——探索性强、杂乱且不可持续。现在的目标是编写更简洁、结构化、可复用的代码。函数、模块、错误处理、脚本编写——这些才是真正能放入数据管道的 Python 代码。

3. **Git and GitHub: Version Control Done Right**
   **Git 和 GitHub：正确的版本控制**
