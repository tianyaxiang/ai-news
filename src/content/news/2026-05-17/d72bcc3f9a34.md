# Why My Coding Assistant Started Replying in Korean When I Typed Chinese
# 为什么我的编程助手在我输入中文时开始用韩语回复？

From a Chinese prompt to a Korean response: an embedding-space investigation into how code vocabulary reshapes language.
从中文提示词到韩语回复：一项关于代码词汇如何重塑语言的嵌入空间研究。

I am Chinese. Primarily, I work with my coding assistant in Chinese. However, my writing is often mixed: many engineering terms are more familiar to me in English (especially terms we use in python, git, etc), and some are even difficult to translate naturally into Chinese.
我是中国人。我主要用中文与编程助手交流。然而，我的表达经常是中英混杂的：许多工程术语我更熟悉英文（尤其是我们在 Python、Git 等中使用的术语），有些术语甚至很难自然地翻译成中文。

Yesterday, I asked my coding assistant in Chinese: “run.py有早停吗？我在恒源云上跑，发现没有触发”, meaning, “Does run.py implement early stopping? I was running the project on a shared GPU service, and I didn’t see early stopping triggered.” As usual, I naturally typed the technical token run.py in its original English form. The model inspected the code and responded with the following:
昨天，我用中文问我的编程助手：“run.py有早停吗？我在恒源云上跑，发现没有触发”。像往常一样，我自然地输入了技术词汇“run.py”。模型检查了代码并回复了以下内容：

（图片：编程助手用韩语回复的截图）

All technical tokens remained in English (run.py, config.py, train_unified), while the explanatory structure shifted into Korean. This is not a unique case. It has happened from time to time: as long as I mixed Chinese and English engineering terms, Korean always appeared.
所有的技术词汇都保持为英文（run.py, config.py, train_unified），而解释性的句子结构却变成了韩语。这不是个例，这种情况时有发生：只要我混合使用中文和英文工程术语，韩语总是会出现。

This made me ask: Is this a language issue, or something deeper in the embedding space?
这让我不禁思考：这是一个语言问题，还是嵌入空间中更深层的问题？

### Hypothesis
### 假设

Embedding spaces are not primarily structured by the nature of languages. Having been trained alongside language models, they tend to be organized by task registers such as academic writing, conversational text, and, in the case of coding assistants, engineering/code.
嵌入空间并非主要按语言本质来构建。由于它们是与语言模型一起训练的，它们往往是按任务语域（task registers）来组织的，例如学术写作、对话文本，而在编程助手的案例中，则是工程/代码语域。

Chinese, although spoken by the largest population in the world, is not a natural medium for the engineering register and has limited representation in technical corpora. In such a context, text may stop behaving like “Chinese” in the embedding space as soon as engineering tokens such as review / branch / commit / PR / diff appear. Instead, it may drift into an engineering attractor field. We will conduct some experiments to provide empirical evidence for this hypothesis.
中文虽然是世界上使用人数最多的语言，但它并不是工程语域的天然媒介，在技术语料库中的代表性也有限。在这种背景下，一旦出现 review / branch / commit / PR / diff 等工程词汇，文本在嵌入空间中可能就不再表现为“中文”，而是漂移进入一个工程吸引子场（engineering attractor field）。我们将进行一些实验，为这一假设提供实证依据。

### Controlled Language Drift
### 受控的语言漂移

We construct the following controlled sequence of sentences where English words take over Chinese ones gradually:
我们构建了以下受控的句子序列，其中英文单词逐渐取代中文单词：

*   Stage 0: 请帮我检查这个分支
*   Stage 1: 请帮我 review 这个分支
*   Stage 2: 请帮我 review 这个 branch
*   Stage 3: Please review this branch pull request commit
*   Stage 4: Please review this branch pull request commit code diff

We now compute similarity using cosine similarity between sentence embeddings. We define Korean and English “clusters” as the average embedding of a small set of representative engineering-related sentences in each language. We use Δ (EN − KO) to denote the difference between English and Korean similarity scores, i.e., Δ = similarity(English) − similarity(Korean).
现在，我们使用句子嵌入之间的余弦相似度来计算相似性。我们将韩语和英语的“聚类”定义为每种语言中一组代表性工程相关句子的平均嵌入。我们使用 Δ (EN − KO) 来表示英语和韩语相似度得分之间的差异，即 Δ = 相似度(英语) − 相似度(韩语)。

| Stage | Korean similarity | English similarity | Δ (EN − KO) |
| :--- | :--- | :--- | :--- |
| 0 | 0.4783 | 0.5141 | 0.0358 |
| 1 | 0.5235 | 0.5728 | 0.0492 |
| 2 | 0.5474 | 0.6140 | 0.0665 |
| 3 | 0.5616 | 0.7314 | 0.1698 |
| 4 | 0.5427 | 0.7398 | 0.1972 |

We observed an interesting phenomenon: Korean similarity increases first and is later overtaken by English similarity. Moreover, the growth in English similarity is non-linear, suggesting a phase-transition–like behavior rather than gradual drift. When projecting the embeddings into two dimensions using PCA, we observe a smooth trajectory in the early stages, followed by a sharp directional jump between Stage 2 and Stage 3, and subsequent stabilization. This pattern indicates that embeddings do not move linearly through space; instead, they appear to transition between attractor basins.
我们观察到一个有趣的现象：韩语相似度先增加，随后被英语相似度超越。此外，英语相似度的增长是非线性的，这表明这是一种类似相变的行为，而不是渐进的漂移。当使用 PCA 将嵌入投影到二维空间时，我们观察到早期阶段轨迹平滑，随后在第 2 阶段和第 3 阶段之间出现剧烈的方向跳跃，之后趋于稳定。这种模式表明嵌入并不是在线性地穿过空间，而是似乎在不同的吸引子盆地（attractor basins）之间转换。

### Real-world Model Behavior
### 现实中的模型行为

Consider again the sentence we mentioned at the beginning. I asked:
再次考虑我们开头提到的句子。我问：

A. “run.py有早停吗？我在恒源云上跑，发现没有触发”
B. “원인을 찾았습니다. 결론: run.py에는 실제로 조기 종료가 없습니다. config.py에 USE_EARLY_STOPPING = True” (in Korean).
C. “我找到了原因。结论：run.py实际上没有早停。config.py里有 USE_EARLY_STOPPING = True。” (Translated back into Chinese)

We compute the similarities of A, B, and C using cosine similarity between sentence embeddings. For comparison, we define three reference clusters: the Chinese cluster as the average embedding of general Chinese natural-language sentences, and the corresponding English and Korean clusters.
我们使用句子嵌入之间的余弦相似度来计算 A、B 和 C 的相似性。为了进行比较，我们定义了三个参考聚类：中文聚类（通用中文自然语言句子的平均嵌入），以及相应的英语和韩语聚类。

| Text | Korean sim | English sim | Chinese sim |
| :--- | :--- | :--- | :--- |
| A. (Chinese prompt) | 0.2003 | 0.2688 | 0.3134 |
| B. (Korean response) | 0.2745 | 0.2983 | 0.1641 |
| C. (Translated Chinese) | 0.1634 | 0.3106 | 0.2798 |

As you can see, translating the Korean response back into Chinese does not send the embedding back to the Chinese region. Instead, it moves even closer to the English clusters. This suggests: Translation could restore language form, but probably not embedding location.
正如你所见，将韩语回复翻译回中文并不能将嵌入带回中文区域。相反，它甚至更接近英语聚类。这表明：翻译可以恢复语言形式，但可能无法恢复嵌入位置。

### Conclusion
### 结论

Both experiments give the same conclusion: the embedding space is not organized by language boundaries. Instead, it is more likely structured by task natures, where engineering English dominates. When a sentence enters this region, language form may change, but the embedding structure remain in the engineering basin, leading to weird behaviors such as replying in Korean even if you are not at all a Korean speaker.
两个实验得出了相同的结论：嵌入空间不是按语言边界组织的。相反，它更有可能是按任务性质组织的，其中工程英语占据主导地位。当一个句子进入这个区域时，语言形式可能会改变，但嵌入结构仍保留在工程盆地中，从而导致一些奇怪的行为，比如即使你根本不会说韩语，模型也会用韩语回复。
