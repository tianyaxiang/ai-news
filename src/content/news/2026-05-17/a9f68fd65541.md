# From Raw Data to Risk Classes
# 从原始数据到风险等级

**Data Science: A practical guide to categorization in credit scoring**

What if your credit scoring model fails not because the algorithm is weak, but because the variables were not prepared in a way the model can properly understand? In credit risk modeling, we often focus on model choice, performance metrics, feature selection, or validation. But before estimating any coefficient, another question deserves attention: how should each variable enter the model?
如果你的信用评分模型失效，原因不是算法太弱，而是变量的准备方式无法被模型正确理解，那该怎么办？在信用风险建模中，我们通常关注模型选择、性能指标、特征选择或验证。但在估计任何系数之前，另一个问题值得关注：每个变量应该如何进入模型？

A raw variable is not always the best representation of risk. A continuous variable may have a non-linear relationship with default. A categorical variable may contain too many modalities. Some variables may include outliers, missing values, unstable distributions, or categories with very few observations. If these issues are ignored, the model may become unstable, difficult to interpret, and less reliable in production.
原始变量并不总是风险的最佳表征。连续变量可能与违约之间存在非线性关系；分类变量可能包含过多的类别；某些变量可能包含异常值、缺失值、不稳定的分布或样本量极少的类别。如果忽视这些问题，模型在生产环境中可能会变得不稳定、难以解释且可靠性降低。

This is where categorization becomes important. Categorization, also called coarse classification, grouping, classing, or binning, consists of transforming raw variable values into a smaller number of meaningful groups. In credit scoring, these groups are not created only for convenience. They are created to make the relationship between the variable and default risk clearer, more stable, and easier to use in a model.
这就是分类（Categorization）变得重要的原因。分类，也称为粗分类、分组、分箱，是指将原始变量值转换为数量较少且有意义的组。在信用评分中，创建这些组不仅仅是为了方便，更是为了使变量与违约风险之间的关系更清晰、更稳定，并更易于在模型中使用。

This step is particularly useful when the final model is a logistic regression, which remains widely used in credit scoring because it is transparent, interpretable, and easy to translate into a scorecard. For categorical variables, categorization helps reduce the number of modalities. For continuous variables, it helps capture non-linear risk patterns, reduce the impact of outliers, handle missing values, improve interpretability, and prepare the variables for Weight of Evidence transformation.
当最终模型为逻辑回归时，这一步骤尤为有用。逻辑回归因其透明、可解释且易于转化为评分卡，在信用评分中仍被广泛使用。对于分类变量，分类有助于减少类别数量；对于连续变量，它有助于捕捉非线性风险模式、减少异常值的影响、处理缺失值、提高可解释性，并为证据权重（Weight of Evidence, WoE）转换做好准备。

### 1. Why categorization is important in credit scoring
### 1. 为什么分类在信用评分中很重要

When building a credit scoring model, variables can be either categorical or continuous. Categorization can be useful for both types of variables, but the motivation is not the same. For categorical variables, the main objective is often to reduce the number of modalities and group categories with similar risk behavior. For continuous variables, the objective is usually to transform a raw numerical scale into a smaller number of ordered risk classes. In both cases, the goal is the same: create variables that are statistically meaningful, economically interpretable, and stable over time.
在构建信用评分模型时，变量可以是分类变量或连续变量。分类对这两种类型的变量都有用，但动机不同。对于分类变量，主要目标通常是减少类别数量，并将具有相似风险行为的类别归为一组。对于连续变量，目标通常是将原始数值尺度转换为数量较少且有序的风险等级。在这两种情况下，目标都是一样的：创建在统计上具有意义、在经济上可解释且随时间保持稳定的变量。

#### 1.1 Categorization Reduces Dimensionality
#### 1.1 分类降低了维度

Let us start with categorical variables. Suppose we have a variable called `industry_sector`, and this variable has 50 different values. If we use this variable directly in a logistic regression model, we need to create dummy variables. Because of collinearity, one category must be used as the reference category. Therefore, for 50 categories, we need: 50−1=49 dummy variables. That means the model must estimate 49 parameters for only one variable. This can quickly become a problem.
让我们从分类变量开始。假设我们有一个名为 `industry_sector`（行业部门）的变量，它有50个不同的值。如果我们直接在逻辑回归模型中使用该变量，我们需要创建虚拟变量（dummy variables）。由于共线性，必须使用一个类别作为参考类别。因此，对于50个类别，我们需要：50-1=49个虚拟变量。这意味着模型必须仅为一个变量估计49个参数。这很快就会成为一个问题。

A categorical variable with too many modalities may lead to unstable coefficients, overfitting, poor robustness, difficulty in interpretation, and higher complexity during monitoring. By grouping similar categories together, we reduce the number of parameters that must be estimated. For example, instead of keeping 50 industry sectors, we may group them into 5 or 6 risk classes. These groups may be based on observed default rates, business expertise, sample size constraints, or a combination of these criteria. The result is a model that is more compact, more stable, and easier to interpret. So, one of the first benefits of categorization is dimension reduction.
具有过多类别的分类变量可能导致系数不稳定、过拟合、鲁棒性差、难以解释以及监控过程中的复杂性增加。通过将相似的类别归为一组，我们减少了必须估计的参数数量。例如，与其保留50个行业部门，我们可能将它们归为5或6个风险等级。这些分组可以基于观察到的违约率、业务专业知识、样本量限制或这些标准的组合。结果是一个更紧凑、更稳定且更易于解释的模型。因此，分类的首要好处之一就是降维。

#### 1.2 Categorization Helps Capture Non-Linear Risk Patterns
#### 1.2 分类有助于捕捉非线性风险模式

For continuous variables, categorization can also be very useful. But before deciding whether to categorize a continuous variable, we should first understand its relationship with default risk. A very simple way to do this is to plot the default rate against the variable. For example, if we have a continuous variable such as `person_income`, we can divide it into several intervals and calculate the default rate in each interval. Then, we plot: the binned values of the variable on the x-axis, the default rate on the y-axis. This allows us to visually inspect the risk pattern.
对于连续变量，分类也非常有用。但在决定是否对连续变量进行分类之前，我们首先应该了解它与违约风险的关系。一种非常简单的方法是将违约率与变量进行绘图。例如，如果我们有一个连续变量，如 `person_income`（个人收入），我们可以将其划分为几个区间，并计算每个区间的违约率。然后，我们将变量的分箱值绘制在X轴上，将违约率绘制在Y轴上。这使我们能够直观地检查风险模式。

If the relationship is monotonic, then the variable already has a clear risk direction. For example: As income increases, default rate decreases. As the loan interest rate increases, the default rate increases. In this case, the relationship is easy to understand. However, if the relationship is non-monotonic, the situation becomes more complex. Suppose default risk decreases for low to medium income levels, but then increases again for very high income levels.
