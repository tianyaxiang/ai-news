### Why perform continual learning?
### 为什么要进行持续学习？

The reason you should perform continual learning is simple: if you’re just using the out-of-the-box version of Claude Code, Codex, or any other coding agent, you’re losing out. Of course, those models are incredibly powerful, and if you compare them to just a few years ago, you can still be many times more efficient than previously. However, that doesn’t really matter. What really matters is that applying continual learning will again provide you with a massive efficiency boost.
你应该进行持续学习的原因很简单：如果你只是使用开箱即用的 Claude Code、Codex 或任何其他编程智能体，你就亏大了。当然，这些模型非常强大，与几年前相比，你已经可以比过去高效数倍。但这并不重要。真正重要的是，应用持续学习将再次为你带来巨大的效率提升。

In this article, I’ll cover one very simple technique on how I make my Claude Code improve itself every single day, and also give you an insight into how I try to optimize my own interactions with Claude Code to make the human coding agent interaction as effective as possible.
在本文中，我将介绍一种非常简单的技术，说明我是如何让我的 Claude Code 每天自我改进的，并分享我对如何优化自身与 Claude Code 交互的见解，以使人机协作尽可能高效。

### Making Claude Code learn from itself
### 让 Claude Code 从自身学习

I’ll start off by covering the simple technique that you can start using right now, which is almost certainly going to improve how your Claude Code performs. You can simply make a skill within Claude Code that goes something like this:
首先，我将介绍一种你现在就可以开始使用的简单技术，它几乎肯定会改善你的 Claude Code 的表现。你只需在 Claude Code 中创建一个类似如下的技能：

> Review my last interactions with Claude Code from the last 24 hours. Look for any problems that I encountered, things that weren't working efficiently, and unnecessary tool calling. Look for common mistakes Claude Code was doing and other things that can be optimized. Look thoroughly through all conversations and make a plan for how we can optimize our flow in the future, both within each repository and cross-repositories. Also look for insights that would be useful for the coding agent to know beforehand, both before entering a repository and when working in multiple repositories at the same time.
> “回顾我过去 24 小时内与 Claude Code 的所有交互。找出我遇到的任何问题、效率低下的环节以及不必要的工具调用。找出 Claude Code 经常犯的错误以及其他可以优化的地方。仔细检查所有对话，并制定一个计划，说明我们未来如何优化工作流程，包括在单个仓库内以及跨仓库的协作。同时，寻找那些对编程智能体提前了解非常有用的见解，无论是进入仓库之前，还是同时处理多个仓库时。”

Let’s say we call this skill `review-past-performance`. Now, what you should do is set up a cron job to trigger this skill at 2 am every night or some time when you know you’re not interacting with your agents actively.
假设我们将此技能命名为 `review-past-performance`。现在，你应该设置一个定时任务（cron job），在每天凌晨 2 点或你确定不会主动与智能体交互的时间触发此技能。

What’s going to happen when you implement and run this skill is that Claude is going to go through all the past conversations that you’ve had over the last 24 hours. It’s going to look at the threads. It’s going to see where you got stuck with Claude Code (i.e., where you spent more time than you should have), and it’s going to see where Claude Code got stuck making incorrect tool calls, incorrect assumptions, or where it simply didn’t have the context it needed to perform the task effectively.
当你实现并运行此技能时，Claude 将会梳理你过去 24 小时内的所有对话。它会查看对话线程，找出你与 Claude Code 协作时卡住的地方（即你花费了不必要时间的地方），并识别 Claude Code 在哪里因为错误的工具调用、错误的假设，或者仅仅是因为缺乏有效执行任务所需的上下文而陷入困境。

It’s then going to make a plan for how to avoid these things from happening in the future and make Claude Code work more effectively in most cases. This will implement changes such as:
然后，它会制定一个计划，说明如何避免未来再次发生这些问题，从而在大多数情况下让 Claude Code 更高效地工作。这将落实诸如以下改进：

*   Adding more information to `agents.md` or similar generic markdown files.
*   在 `agents.md` 或类似的通用 Markdown 文件中添加更多信息。
*   Creating specific skills that the agent can either load on demand or run on demand when dealing with certain tasks.
*   创建特定的技能，供智能体在处理特定任务时按需加载或运行。
*   Implementing specific scripts or tooling, such as pre-commit hooks, testing scripts, and similar, to avoid mistakes from happening again.
*   实施特定的脚本或工具（如 pre-commit hooks、测试脚本等），以防止错误再次发生。

The best part about setting up a cron job to run this skill on a daily basis is that you don’t even have to interact with agents at all. It’s going to be able to self-reflect in an efficient manner, discover inefficiencies, tweak them, and thus improve Claude Code over time.
设置每日定时任务来运行此技能最棒的地方在于，你根本不需要亲自与智能体交互。它能够以高效的方式进行自我反思、发现低效之处、进行调整，从而随着时间的推移不断改进 Claude Code。
