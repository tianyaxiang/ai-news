# TeamTR: Trust-Region Fine-Tuning for Multi-Agent LLM Coordination
# TeamTR：用于多智能体大模型协作的信任域微调

**Abstract:** Multi-agent LLM systems have shown promise for complex reasoning, yet recent evaluations reveal they often underperform single-model baselines. We identify a structural failure mode in sequential fine-tuning of shared-context teams: updating one agent shifts the team's context distribution, and when subsequent updates are evaluated on cached rollouts, this mismatch compounds. 

**摘要：** 多智能体大模型（LLM）系统在复杂推理方面展现出了潜力，但近期的评估显示，它们的表现往往不如单模型基准。我们发现共享上下文团队在顺序微调过程中存在一种结构性失效模式：更新一个智能体会改变团队的上下文分布，而当后续更新在缓存的推演（rollouts）上进行评估时，这种不匹配会不断累积。

We formalize this as the compounding occupancy shift and prove that stale-occupancy evaluation incurs a penalty that scales quadratically with the number of agents. In contrast, intermediate-occupancy evaluation reduces this to linear scaling. 

我们将此现象形式化为“复合占用偏移”（compounding occupancy shift），并证明基于陈旧占用的评估会产生随智能体数量呈二次方增长的惩罚。相比之下，中间占用评估将这一惩罚降低到了线性增长水平。

We propose TeamTR, a trust-region framework that resamples trajectories after each component update and enforces per-agent divergence control, yielding rigorous per-update and per-stage improvement lower bounds. 

我们提出了 TeamTR，这是一个信任域框架，它在每次组件更新后重新采样轨迹，并强制执行针对每个智能体的发散控制，从而为每次更新和每个阶段的改进提供了严格的下界。

Experiments show that TeamTR outperforms single-agent and sequential baselines with 7.1% on average, mitigates coordination regressions, and supports plug-and-play component replacement. Code is available at this https URL.

实验表明，TeamTR 的表现平均比单智能体和顺序微调基准高出 7.1%，减轻了协作回归问题，并支持即插即用的组件替换。代码可在该链接获取。
