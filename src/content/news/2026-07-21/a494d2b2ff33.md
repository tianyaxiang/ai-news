# Automatizar una distribuidora con IA en Argentina: los procesos reales y el stack que uso
# 在阿根廷利用人工智能实现分销商自动化：实际流程与我使用的技术栈

Trabajé más de 20 años en una distribuidora antes de dedicarme a la inteligencia artificial. Cuando alguien me pregunta "¿la IA sirve para una distribuidora?", la respuesta corta es: es de los rubros donde más se nota, porque el cuello de botella casi nunca es vender — es la montaña de tarea administrativa detrás de cada venta. Esta es una guía concreta de qué se puede automatizar hoy, en 2026, en una pyme argentina real, y con qué stack lo armo.
在投身人工智能领域之前，我在一家分销商工作了超过 20 年。当有人问我“人工智能对分销商有用吗？”时，简短的回答是：这是最能体现其价值的行业之一，因为瓶颈几乎从来不是销售，而是每笔销售背后堆积如山的行政工作。这是一份关于 2026 年在阿根廷的真实中小企业中，目前可以实现哪些自动化以及我使用什么技术栈的实操指南。

El stack, en una línea: Nada exótico ni caro: n8n (self-hosted) como orquestador de los flujos. Un LLM (Claude / GPT) para las partes que requieren interpretar texto o archivos con formato variable. Conexión por API a lo que el negocio ya usa: WhatsApp, el sistema de gestión, ARCA/AFIP, Google Sheets/DB. El costo de la IA por operación suele ser de centavos. Lo que se paga es el armado y el mantenimiento, no un ERP de seis cifras.
技术栈概括：没有异国情调或昂贵的工具。使用 n8n（自托管）作为流程编排器；使用大语言模型（Claude / GPT）处理需要解读文本或格式多变文件的部分；通过 API 连接到企业现有的系统：WhatsApp、管理系统、ARCA/AFIP（阿根廷税务局）、Google Sheets/数据库。人工智能的单次操作成本通常仅为几分钱。你支付的是搭建和维护费用，而不是六位数的 ERP 系统费用。

1. Actualización de listas de precios (el clásico que a todos les duele)
El proveedor manda la lista nueva en un PDF o Excel con formato distinto cada vez, y alguien carga cientos de artículos a mano. Horas, con errores, y cuando terminás ya llegó otra lista.
1. 更新价格表（每个人都头疼的经典问题）
供应商每次发送的 PDF 或 Excel 格式的价目表都不一样，导致有人必须手动录入数百个条目。这不仅耗时且容易出错，而且当你录完时，新的价目表又来了。

El flujo: Archivo del proveedor (PDF/Excel) | Parser + LLM -> normaliza {codigo, descripcion, precio} | (aunque cambie el formato de origen) | Validacion (codigo existe? variacion de precio razonable?) | Update al sistema de gestion via API.
流程：供应商文件 (PDF/Excel) | 解析器 + LLM -> 标准化 {代码, 描述, 价格} | （即使原始格式发生变化） | 验证（代码是否存在？价格变动是否合理？） | 通过 API 更新到管理系统。

La clave es usar el LLM solo para la parte difícil (interpretar un layout que cambia), no para hacer la matemática. Media jornada pasa a minutos. Es probablemente el proceso con mejor retorno, y casi nadie lo ofrece porque no es tan "vistoso" como un chatbot.
关键在于仅将 LLM 用于困难部分（解读不断变化的布局），而不是进行数学计算。半天的工作量缩短为几分钟。这可能是回报率最高的流程，但几乎没人提供这项服务，因为它不像聊天机器人那样“显眼”。

2. Pedidos por WhatsApp que se cargan solos
Tus clientes ya piden por WhatsApp; el problema es que una persona transcribe cada pedido al sistema. Un agente de IA lee el mensaje ("mandame 10 cajas de X y 5 de Y"), lo estructura, valida contra stock y lo deja listo para confirmar. El vendedor deja de ser transcriptor y vuelve a vender.
2. 自动录入的 WhatsApp 订单
你的客户已经在通过 WhatsApp 下单了；问题在于需要专人将每个订单转录到系统中。人工智能代理会读取消息（“给我发 10 箱 X 和 5 箱 Y”），将其结构化，根据库存进行验证，并准备好等待确认。销售人员不再是转录员，而是回归到销售本职。

Técnicamente es una tool call: el mensaje libre entra, el modelo devuelve un objeto de pedido tipado (items[], cantidades, cliente), y ese objeto se valida antes de tocar el sistema.
从技术上讲，这是一个工具调用（tool call）：自由文本消息进入，模型返回一个类型化的订单对象（项目[]、数量、客户），该对象在触及系统之前会经过验证。

3. Remitos y facturación automáticos
Al confirmar un pedido, el comprobante se genera solo: valida datos del cliente, emite la factura electrónica contra ARCA/AFIP, la manda por mail y WhatsApp, y actualiza el sistema contable. Sin copiar datos de una pantalla a otra.
3. 自动发货单与开票
确认订单后，凭证会自动生成：验证客户数据，向 ARCA/AFIP 开具电子发票，通过邮件和 WhatsApp 发送，并更新会计系统。无需在不同屏幕之间复制数据。

4. Conciliación de stock
La eterna diferencia entre "lo que dice el sistema" y "lo que hay en el depósito". Un flujo que cruza entradas, salidas y ajustes, y avisa cuando algo no cuadra — antes de quedarte sin mercadería o comprar de más.
4. 库存核对
“系统显示”与“仓库实际”之间永远存在差异。通过一个流程交叉比对入库、出库和调整记录，并在出现不匹配时发出警报——防止缺货或过度采购。

Dos consejos de ingeniería antes de automatizar nada:
1. No automatices un proceso que está roto. Si tu carga de precios es un caos, primero ordenala. Automatizar el caos solo lo hace más rápido. La IA multiplica lo que ya tenés, para bien o para mal.
2. Validá siempre la salida del LLM antes de escribir en un sistema real. El modelo interpreta; tu código decide. Un precio con una variación del 400% o un código inexistente no deberían llegar nunca a producción sin un check determinístico en el medio.
在进行任何自动化之前的两个工程建议：
1. 不要自动化一个已经损坏的流程。如果你的价格录入是一团糟，先整理好它。自动化混乱只会让混乱发生得更快。人工智能会放大你现有的东西，无论好坏。
2. 在写入真实系统之前，务必验证 LLM 的输出。模型负责解读，你的代码负责决策。如果价格变动达到 400% 或代码不存在，在没有中间确定性检查的情况下，绝不应进入生产环境。

Por dónde empezar:
No intentes automatizar todo de una. Elegí el proceso que más horas te come por semana —en una distribuidora casi siempre es la carga de precios o la de pedidos— y arrancá por ahí. Medí cuántas horas te devuelve. Con ese primer resultado, el resto se justifica solo.
从哪里开始：
不要试图一次性自动化所有事情。选择每周占用你时间最多的流程——在分销商中，通常是价格录入或订单录入——从那里开始。衡量它为你节省了多少小时。有了第一个成果，其余的自动化就顺理成章了。
