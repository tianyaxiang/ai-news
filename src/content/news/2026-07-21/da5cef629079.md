# How to Run Claude Code Agents for 24+ Hours
# 如何让 Claude 代码智能体运行超过 24 小时

In this article, I’ll discuss how to have your coding agents from Codex and Claude Code run for longer periods of time, allowing you to run more agents in parallel.
在本文中，我将探讨如何让你的 Codex 和 Claude Code 编程智能体运行更长时间，从而使你能够并行运行更多的智能体。

One of the keys to being able to run a lot of coding agents in parallel is to ensure you have long-running sessions. This means sessions that are interrupted as little as possible. For example, asking for human input, permissions and so on.
能够并行运行大量编程智能体的关键之一，是确保拥有长时间运行的会话。这意味着会话应尽可能少地被中断，例如减少对人工输入、权限确认等操作的请求。

Frontier coding agents are getting better and better at running for longer periods of time, being more eager to complete the goals that they receive from the human input. I especially notice this from the latest Claude Fable model and GPT-5.6.
前沿编程智能体在长时间运行方面的表现越来越好，它们更渴望完成人类输入的目标。我从最新的 Claude Fable 模型和 GPT-5.6 中尤其感受到了这一点。

However, the agents in themselves don’t solve everything. You also need to have a harness and structure around your coding that ensures the agents are able to run for longer periods of time.
然而，智能体本身并不能解决所有问题。你还需要在编程环境中建立一套框架和结构，以确保智能体能够持续运行更长时间。

In this article, I’ll discuss: How to create an environment where agents can run for longer periods of time; How to give them the permissions they need; How to ensure agents verify their own work.
在本文中，我将讨论：如何创建智能体可以长时间运行的环境；如何赋予它们所需的权限；以及如何确保智能体能够验证自己的工作。

### Why run long-running agents
### 为什么要运行长时间的智能体

First of all, I’d like to cover why you should follow the tips within this article. In this instance, it’s about why you should run agents for a longer period of time. You might think that you want agents to run fast and get work done as quickly as possible. Of course, to a certain extent, you want stuff done as quickly as possible, but there’s also a trade-off here with how much a human has to interact with the agent to get the work done.
首先，我想说明为什么要遵循本文中的建议。具体来说，就是为什么要让智能体运行更长时间。你可能认为希望智能体运行得越快越好，尽快完成工作。当然，在一定程度上，你确实希望任务完成得越快越好，但这与人类为了完成工作而必须与智能体进行的交互频率之间存在权衡。

After the release of coding agents, and especially after they became very powerful, after the release of Claude Opus 4.5, the bottleneck within coding has become the human review time; i.e., it’s not actually implementing the code that takes time; it’s more reviewing the results of the code, checking that it’s implemented correctly and according to the specs, that takes time.
在编程智能体发布之后，尤其是 Claude Opus 4.5 发布并展现出强大能力后，编程的瓶颈已转变为人工审核时间；也就是说，耗时的并不是代码实现本身，而是审核代码结果、检查其是否正确实现并符合规范的过程。

Thus, since this is the bottleneck, you want to minimize how much time you spend on review, which is why you want the agents to run for longer periods of time, implementing everything and reviewing it themselves before handing it off to the human. This minimizes the time a human has to spend verifying a task, which is ideal because it is the bottleneck currently within programming.
因此，既然这是瓶颈所在，你就需要尽量减少花在审核上的时间。这就是为什么你希望智能体运行更长时间，在交付给人类之前，让它们自行完成所有实现并进行自我审核。这最大限度地减少了人类验证任务所需的时间，这是理想的状态，因为这正是目前编程中的瓶颈。

### How to run agents for over 24 hours
### 如何让智能体运行超过 24 小时

Now I’ll move on to some more specific techniques and tips on how to actually run agents for longer periods of time. As a high-level overview, I’ll cover these techniques and tips: Give agents all permissions; Give agents the ability to verify their own work; Make agents review the code (Codex); Running agents remotely.
现在，我将介绍一些更具体的技术和技巧，说明如何让智能体运行更长时间。概括来说，我将涵盖以下技术和建议：赋予智能体所有权限；赋予智能体自我验证工作的能力；让智能体审核代码（Codex）；远程运行智能体。

#### Permissions
#### 权限

The first thing you can just start with right away is to make sure that the agents have all the permissions that they need. If an agent has to stop asking for permissions, you’re wasting a lot of time, and in almost all cases, it’s just something you accept anyway.
你可以立即着手的第一件事，是确保智能体拥有它们所需的所有权限。如果智能体必须不断停下来请求权限，你就是在浪费大量时间，而且在几乎所有情况下，这些权限最终都是你需要授予的。

There are numerous ways to do this, and you can have auto-mode or skip all permissions, which are settings available in both Codex and Claude Code. It could be worth noting that you should be running in sandbox mode so that you protect your computer and your data and make sure nothing gets deleted permanently.
实现这一点的方法有很多，你可以开启自动模式或跳过所有权限确认，这些设置在 Codex 和 Claude Code 中都可用。值得注意的是，你应该在沙盒模式下运行，以保护你的计算机和数据，确保没有任何内容被永久删除。

#### Verify work
#### 验证工作

Another very important point is to ensure the agents can verify their own work. This is a topic I’ve discussed before in several other articles, but it all comes down to how you describe a task to your agent and which tools you give the agents.
另一个非常重要的点是确保智能体能够验证自己的工作。这是我在其他几篇文章中讨论过的话题，但归根结底，这取决于你如何向智能体描述任务，以及你赋予了智能体哪些工具。

When you describe a task to the agent, you need to make sure you give as detailed instructions as possible, actually mentioning what you want to achieve, but most importantly, you have to tell the agent how to know when it’s completed its work. And the best way to do this is typically to tell the agent how to test the implementation.
当你向智能体描述任务时，必须确保提供尽可能详细的说明，明确指出你想要实现的目标，但最重要的是，你必须告诉智能体如何判断工作已完成。实现这一点的最佳方法通常是告诉智能体如何测试其实现结果。
