图1：图模型为不确定性提供了结构：贝叶斯网络使用有向依赖关系，马尔可夫网络使用无向兼容性，而马尔可夫逻辑则在对象和关系之上增加了加权规则。

**Most machine learning explanations begin with prediction. A churn model estimates whether a customer is likely to leave. A fraud model estimates whether a transaction is suspicious. A diagnosis model estimates the likelihood of a condition from symptoms, tests and history. A document classifier assigns a category from text, metadata or embeddings.**
大多数机器学习的解释都始于预测。流失模型估计客户是否可能流失；欺诈模型估计交易是否可疑；诊断模型根据症状、检查和病史估计某种疾病的可能性；文档分类器则根据文本、元数据或嵌入向量分配类别。

**In each case, the setup is broadly the same. We have some observed information, usually called the inputs or features, and a thing we want to predict, usually called the target. If we write the inputs as X and the target as Y, then the usual supervised learning problem is to learn a model of: P(Y | X)**
在每种情况下，设置大体相同。我们拥有一些观测信息（通常称为输入或特征），以及我们想要预测的事物（通常称为目标）。如果我们用 X 表示输入，用 Y 表示目标，那么常见的监督学习问题就是学习一个模型：P(Y | X)

**That is, given the inputs, how likely is each possible output? This framing is very useful. It gives us logistic regression, support vector machines, random forests, gradient boosting, neural networks, and a large part of modern applied machine learning. In many production settings, it is the right starting point.**
也就是说，给定输入，每个可能的输出有多大概率？这种框架非常有用。它衍生出了逻辑回归、支持向量机、随机森林、梯度提升、神经网络以及现代应用机器学习的很大一部分。在许多生产环境中，这是正确的起点。

**You have a target, some data, and a metric. The model’s job is to produce a good answer for that target.**
你有一个目标、一些数据和一个指标。模型的工作就是为该目标提供一个好的答案。

**Bayesian networks come from a different modelling instinct. They are useful when we want to represent a small uncertain world rather than only learn a mapping from inputs to one output. Instead of dividing the world into “inputs” and “target”, we describe a collection of uncertain variables: X₁, …, Xₙ**
贝叶斯网络源于一种不同的建模直觉。当我们想要表示一个微小的不确定世界，而不仅仅是学习从输入到单一输出的映射时，它们非常有用。我们不再将世界划分为“输入”和“目标”，而是描述一组不确定变量：X₁, …, Xₙ

**Some of these variables may be observed, some may be hidden, and any of them might become the thing we care about predicting. The graph describes how these uncertain variables depend on one another, where evidence can enter, and how beliefs should update when new information arrives.**
其中一些变量可能是已观测的，一些可能是隐藏的，任何一个都可能成为我们想要预测的对象。图模型描述了这些不确定变量如何相互依赖、证据从何处进入，以及当新信息到来时信念应如何更新。

**A classifier is usually built around one prediction question: Given these inputs, what is the output? A Bayesian network is built around a more general uncertainty question: These uncertain things are related. Given evidence anywhere in the system, what should we now believe about the rest?**
分类器通常围绕一个预测问题构建：给定这些输入，输出是什么？而贝叶斯网络则围绕一个更通用的不确定性问题构建：这些不确定事物是相关的。给定系统中任何地方的证据，我们现在应该如何看待其余部分？

**So rather than only asking for P(Y | X), we represent a fuller joint distribution over the variables: P(X₁, …, Xₙ)**
因此，我们不再仅仅询问 P(Y | X)，而是表示变量上更完整的联合分布：P(X₁, …, Xₙ)

**The purpose of the graph is to make that joint distribution manageable by encoding which variables depend directly on which others. We will build the intuition in stages: first the full joint distribution, then a tiny wet-grass Bayesian network in Python, then explaining away any conditional independence, then parameter learning and inference, and finally Markov networks and Markov logic.**
图的目的在于通过编码哪些变量直接依赖于其他变量，使联合分布变得可控。我们将分阶段建立直觉：首先是完整的联合分布，然后是一个 Python 实现的微型“湿草”贝叶斯网络，接着解释条件独立性，然后是参数学习与推理，最后是马尔可夫网络和马尔可夫逻辑。

**The full joint distribution is the thing we cannot afford. Imagine a small world with a few uncertain facts: cloudiness, rain, sprinkler use, wet grass, slippery pavement, traffic, and late arrival. In principle, the most complete probabilistic model would describe the probability of every possible configuration: P(Cloudy, Rain, Sprinkler, WetGrass, SlipperyPavement, Traffic, LateArrival)**
完整的联合分布是我们无法负担的。想象一个小世界，其中包含几个不确定的事实：多云、下雨、喷水器使用、草地湿润、路面湿滑、交通状况和迟到。原则上，最完整的概率模型将描述每种可能配置的概率：P(多云, 下雨, 喷水器, 草地湿润, 路面湿滑, 交通, 迟到)

**This is the full joint distribution. If we had it, we could ask almost anything: P(Rain | WetGrass), P(LateArrival | Rain, Traffic), P(Sprinkler | WetGrass, no Rain).**
这就是完整的联合分布。如果我们拥有它，我们几乎可以询问任何问题：P(下雨 | 草地湿润)、P(迟到 | 下雨, 交通)、P(喷水器 | 草地湿润, 未下雨)。

**The problem is that the full joint distribution grows brutally fast. If each variable is binary, then n variables require 2ⁿ possible configurations. Seven variables already give 128 states. Twenty variables give more than a million. One hundred variables are out of reach.**
问题在于，完整的联合分布增长得极其迅速。如果每个变量都是二元的，那么 n 个变量就需要 2ⁿ 种可能的配置。七个变量就已经有 128 种状态。二十个变量则超过一百万种。一百个变量则是遥不可及的。

**So the problem is partly statistical and partly representational. We need some way to avoid treating every variable as potentially entangled with every other variable in every possible way. This is where graphical models enter.**
因此，这个问题既是统计学问题，也是表征问题。我们需要某种方法来避免将每个变量都视为可能以各种方式与所有其他变量纠缠在一起。这就是图模型发挥作用的地方。

**A graphical model is a compact way of representing a probability distribution by making assumptions about structure. The graph says which local relationships we model directly, and which relationships we treat as indirect consequences of those local pieces.**
图模型是一种通过对结构做出假设来紧凑表示概率分布的方法。图模型指明了我们直接建模的局部关系，以及我们将哪些关系视为这些局部片段的间接后果。

**The central idea is simple: Do not model the whole uncertain world as one giant table. Break it into smaller conditional pieces.**
核心思想很简单：不要将整个不确定世界建模为一个巨大的表格，而是将其分解为更小的条件片段。

**A Bayesian network is a directed map of local dependence. Take the classic wet grass example. You walk outside and see that the grass is wet. Maybe it rained. Maybe the sprinkler was on. Maybe both happened. The model says that rain and sprinkler both influence whether the grass is wet. The probability distribution factorises as: P(Rain, Sprinkler, WetGrass) = P(Rain) P(Sprinkler) P(WetGrass | Rain, Sprinkler)**
贝叶斯网络是局部依赖的有向图。以经典的“湿草”为例。你走到户外，看到草地湿了。可能是下雨了，可能是喷水器开了，也可能两者都发生了。该模型认为，下雨和喷水器都影响草地是否湿润。概率分布分解为：P(下雨, 喷水器, 草地湿润) = P(下雨) P(喷水器) P(草地湿润 | 下雨, 喷水器)

**Instead of writing one table for every combination of rain, sprinkler and wet grass, we write smaller pieces: P(Rain), P(Sprinkler), P(WetGrass | Rain, Sprinkler). The conditional table for wet grass is easy to imagine: Each variable gets a local probability model conditioned on its parents. For a Bayesian network over X₁, …, Xₙ, the general factorisation is: P(X₁, …, Xₙ) = product over i of P(Xᵢ | Parents(Xᵢ))**
我们不再为下雨、喷水器和草地湿润的每一种组合编写一个表格，而是编写更小的片段：P(下雨)、P(喷水器)、P(草地湿润 | 下雨, 喷水器)。草地湿润的条件表很容易想象：每个变量都获得一个以其父节点为条件的局部概率模型。对于覆盖 X₁, …, Xₙ 的贝叶斯网络，其通用分解为：P(X₁, …, Xₙ) = Π P(Xᵢ | Parents(Xᵢ))

**Each variable only needs to know about its direct parents. That is how a large probabilistic model becomes a collection of smaller, understandable pieces.**
每个变量只需要了解其直接父节点。这就是大型概率模型如何成为一系列更小、更易理解的片段的方法。

**A tiny Bayesian network in Python. It helps to make this concrete. We will build the smallest useful version of the wet grass model: rain may or may not happen, the sprinkler may or may not be on, the grass may or may not be wet. The structure is simple. Rain and sprinkler both influence whether the grass is wet. We can write the model as: P(Rain, Sprinkler, WetGrass) = P(Rain) P(Sprinkler) P(WetGrass | Rain, Sprinkler). The first two terms are simple prior probabilities. The final term is a conditional probability table. Here is a deliberately small implementation.**
Python 中的微型贝叶斯网络。将其具体化会有所帮助。我们将构建“湿草”模型中最小的有用版本：可能下雨也可能不下雨，喷水器可能开启也可能关闭，草地可能湿润也可能干燥。结构很简单。下雨和喷水器都影响草地是否湿润。我们可以将模型写为：P(下雨, 喷水器, 草地湿润) = P(下雨) P(喷水器) P(草地湿润 | 下雨, 喷水器)。前两项是简单的先验概率。最后一项是条件概率表。以下是一个特意简化的实现。
