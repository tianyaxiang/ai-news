# Run a Local LLM with OpenClaw on Your Mac Mini
# 在 Mac Mini 上通过 OpenClaw 运行本地大语言模型 (LLM)

Tired of your monthly API bill? Follow this tested guide to set up a high-performance local LLM on your Mac Mini without the headaches.
厌倦了每月的 API 账单？请跟随这篇经过验证的指南，在你的 Mac Mini 上轻松配置高性能的本地大语言模型，告别烦恼。

You bought the Mac Mini for Openclaw. Perfect. Unfortunately, as of late, Anthropic has pushed OpenClaw users toward its pay-per-token API, turning what was once a one-time hardware purchase into a (large) ongoing expense. Even if you use OpenAI, you’re still going to be paying quite a bit monthly.
你买了 Mac Mini 是为了运行 OpenClaw，这很棒。但不幸的是，最近 Anthropic 将 OpenClaw 用户推向了按 Token 付费的 API 模式，将原本的一次性硬件投入变成了持续的（且不菲的）开支。即使你使用 OpenAI，每月也依然需要支付不少费用。

💵💵 Running a local model eliminates the monthly cost for your OpenClaw agents, entirely. 💵💵 However, getting everything installed and configured can be confusing, especially if you’re new to local LLMs. In this article, I’ll show you how to set up a local LLM (in the most pain-free way) on your Mac Mini that can power your agent for free. You can use it even if you’re a beginner.
💵💵 运行本地模型可以完全消除 OpenClaw 代理的月度成本。💵💵 然而，安装和配置过程可能会让人困惑，尤其是如果你是本地 LLM 的新手。在本文中，我将向你展示如何在 Mac Mini 上以最无痛的方式设置本地 LLM，从而免费驱动你的代理。即使你是初学者也能轻松上手。

🤨 “I’ve heard that local LLMs don’t work as well, is that true?” A local LLM (properly set up) will perform almost indistinguishably for tasks like emails, calendar management, reminders, home IoT automation and basic internet research (things you actually do with OpenClaw). If you need to do something more advanced, like using OpenClaw for software engineering, there’s a link at the bottom which highlights how to set up a fallback model.
🤨 “我听说本地 LLM 的效果没那么好，是真的吗？”如果配置得当，本地 LLM 在处理电子邮件、日程管理、提醒、家庭物联网自动化和基础网络搜索（这些正是你使用 OpenClaw 的日常任务）时，表现几乎与云端模型无异。如果你需要进行更高级的操作，例如使用 OpenClaw 进行软件工程，文末提供了设置备用模型的链接。

### Setting things up
### 环境配置

First, install OpenClaw using the official guide. If you’ve already done this, skip this step.
首先，请按照官方指南安装 OpenClaw。如果你已经完成此步骤，请跳过。

#### 1. Install llama.cpp
#### 1. 安装 llama.cpp

We’re going to skip using Ollama (the recommended local provider), and opt for llama.cpp. By using a quantized model along with llama.cpp, we can speed up inference by as much as 70%. We need to build llama.cpp from the source with metal flags on and cuda off. This handles some of the optimizations needed to run the model on your Mac at full speed. Simply follow the steps below.
我们将跳过 Ollama（官方推荐的本地提供程序），改用 llama.cpp。通过将量化模型与 llama.cpp 结合使用，我们可以将推理速度提升高达 70%。我们需要从源码构建 llama.cpp，开启 Metal 加速并关闭 CUDA。这能处理在 Mac 上全速运行模型所需的优化。只需按照以下步骤操作即可。

1️⃣ First, from your home directory, install some prerequisites using brew.
1️⃣ 首先，在你的主目录下，使用 brew 安装一些先决条件。
```bash
$ brew install cmake curl
```

2️⃣ Then, build llama.cpp with the appropriate flags.
2️⃣ 然后，使用相应的标志构建 llama.cpp。
```bash
# Clone llama.cpp
git clone https://github.com/ggml-org/llama.cpp

# Configure build with Metal acceleration
cmake llama.cpp -B llama.cpp/build \
  -DBUILD_SHARED_LIBS=OFF \
  -DGGML_METAL=ON \
  -DGGML_CUDA=OFF

# Build
cmake --build llama.cpp/build \
  --config Release \
  -j$(sysctl -n hw.ncpu) \
  --clean-first \
  --target llama-cli llama-mtmd-cli llama-server llama-gguf-split
```
Now, we have llama.cpp available to use.
现在，我们已经可以使用 llama.cpp 了。

#### 2. Download the local LLM
#### 2. 下载本地 LLM

As mentioned, the key to getting good performance from a local model is quantization. Quantizing allows us to use a larger, more capable model, “compressed” intelligently so that it fits on smaller hardware. This allows the quantized model to retain most of the performance of its full-size source model. Unless you have a large GPU or Mac with the maximum amount of unified memory (80GB+ VRAM), quantizing is a must.
如前所述，从本地模型获得良好性能的关键在于量化。量化使我们能够使用更大、能力更强的模型，通过智能“压缩”使其适配较小的硬件。这使得量化模型能够保留其原始全尺寸模型的大部分性能。除非你拥有大型 GPU 或配备最大统一内存（80GB+ VRAM）的 Mac，否则量化是必不可少的。

Blindly following the OpenClaw documentation while trying to use a quantized model will leave you confused and frustrated. There is simply no guide available which clearly outlines how to make quantized models work with agents. Below is a tested recipe that will work for your agent.
盲目遵循 OpenClaw 文档并尝试使用量化模型会让你感到困惑和沮丧。目前没有任何指南清楚地说明如何让量化模型与代理协同工作。以下是一个经过验证的、适用于你代理的方案。

**Model Choice: Qwen 3.5-9B**
**模型选择：Qwen 3.5-9B**

Here we’re using Qwen 3.5 (the 9B parameter version). As of June 2026, it’s a top performer for local models, edging out Gemma 4-12B. This will fit on both a 16GB or 24GB Mac with a total of 6-8GB of RAM required. Users also rank this highly for OpenClaw. Also remember that agents require longer contexts, which will prevent us from running a larger 27B version, even with quantization.
这里我们使用 Qwen 3.5（9B 参数版本）。截至 2026 年 6 月，它是本地模型中的佼佼者，略胜于 Gemma 4-12B。它可以在 16GB 或 24GB 的 Mac 上运行，总共需要 6-8GB 内存。用户对它在 OpenClaw 上的表现评价很高。还要记住，代理需要更长的上下文，这使得我们即使使用量化也无法运行更大的 27B 版本。

1️⃣ Let’s download the model
1️⃣ 下载模型
```bash
curl -L -o models/Qwen3.5-9B-UD-Q4_K_KL.gguf \
"https://huggingface.co/unsloth/Qwen3.5-9B-MTP-GGUF/resolve/main/Qwen3.5-9B-UD-Q4_K_XL.gguf?download=true"
```

2️⃣ Download the template, save it to templates.
2️⃣ 下载模板并保存到 templates 文件夹。
```bash
mkdir templates && \
curl -o templates/qwen35.jinja \
"https://huggingface.co/froggeric/Qwen-Fixed-Chat-Templates/resolve/main/chat_template.jinja"
```
Important, you must use an agent compatible template for OpenClaw. Without this step, nothing will work.
重要提示：你必须为 OpenClaw 使用兼容的代理模板。没有这一步，一切都无法运行。

#### 3. Start llama-server
#### 3. 启动 llama-server

Llama-server will serve as our backend API. OpenClaw will use this webservice instead of calling the API from OpenAI or Anthropic directly. We installed llama-server already, and downloaded our model. Let’s run a quick test.
Llama-server 将作为我们的后端 API。OpenClaw 将使用此 Web 服务，而不是直接调用 OpenAI 或 Anthropic 的 API。我们已经安装了 llama-server 并下载了模型，现在进行快速测试。

1️⃣ Run a quick test
1️⃣ 运行快速测试
```bash
./llama.cpp/llama-server \
  -m models/Qwen3.5-9B-UD-Q4_K_XL.gguf \
  --chat-template-file templates/qwen35.jinja \
  --temp 0.7 \
  --top-p 0.9 \
  --top-k 20 \
  -c 64000 \
  -ngl 20 \
  --host 127.0.0.1 \
  --port 8080
```
You should see something like so (without errors):
你应该看到类似以下内容（且没有错误）：
`srv llama_server: model loaded`
`llama_server: server is listening on http://127.0.0.1:8080`
`update_slots: all slots are idle`
